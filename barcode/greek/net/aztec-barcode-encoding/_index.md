# Κωδικοποίηση Γραμμωτού Κώδικα Aztec

Είστε έτοιμοι να βυθιστείτε στον κόσμο της κωδικοποίησης γραμμωτού κώδικα Aztec και να μάθετε πώς να **δημιουργήσετε γραμμωτό κώδικα Aztec** με το Aspose.BarCode για .NET; Αυτή η βιβλιοθήκη σας δίνει πλήρη έλεγχο στο μέγεθος, τη διόρθωση σφαλμάτων και την αναπαράσταση δεδομένων, καθιστώντας την ιδανική για όλα, από την κινητή έκδοση εισιτηρίων μέχρι την παρακολούθηση αποθεμάτων.

## Σύντομες Απαντήσεις
- **Ποια βιβλιοθήκη υποστηρίζει γραμμωτούς κώδικες Aztec;** Aspose.BarCode for .NET  
- **Μπορώ να αλλάξω την αναλογία διαστάσεων;** Yes, via the `AspectRatio` property  
- **Είναι δυνατή η κωδικοποίηση σε επίπεδο byte;** Absolutely – use the `EncodeBytes` method  
- **Ποια επίπεδα διόρθωσης σφαλμάτων είναι διαθέσιμα;** Levels 0 to 4 (higher = more redundancy)  
- **Χρειάζομαι άδεια για παραγωγή;** Yes, a commercial license removes evaluation limits  

## Τι είναι ένας γραμμωτός κώδικας Aztec;
Ένας γραμμωτός κώδικας Aztec είναι ένας δισδιάστατος κώδικας πλέγματος που μπορεί να κωδικοποιήσει έως 3.000 αριθμητικούς ή 2.300 αλφαριθμητικούς χαρακτήρες. Διαθέτει ένα κεντρικό μοτίβο εντοπισμού, επιτρέποντας γρήγορη σάρωση ακόμη και όταν το σύμβολο εκτυπώνεται σε χαμηλή ανάλυση. Επειδή το μοτίβο βρίσκεται στο κέντρο, ο κώδικας μπορεί να διαβαστεί από οποιαδήποτε κατεύθυνση, καθιστώντας τον εξαιρετικά αξιόπιστο για κινητές και βιομηχανικές εφαρμογές.

## Πώς προσαρμόζετε την αναλογία διαστάσεων ενός γραμμωτού κώδικα Aztec;
`BarcodeGenerator` είναι η κύρια κλάση που χρησιμοποιείται για τη δημιουργία γραμμωτών κωδίκων στο Aspose.BarCode. Ορίστε την ιδιότητα `AspectRatio` στο αντικείμενο `BarcodeGenerator` στην επιθυμητή αναλογία πλάτους‑προς‑ύψος, στη συνέχεια δημιουργήστε την εικόνα. Η ρύθμιση της αναλογίας διαστάσεων βοηθά στην προσαρμογή του γραμμωτού κώδικα σε περιορισμένους χώρους UI ή διατάξεις ετικετών χωρίς να θυσιάζεται η αξιοπιστία της σάρωσης, και επίσης σας επιτρέπει να ταιριάξετε τις οδηγίες branding ή τις συγκεκριμένες απαιτήσεις του εκτυπωτή.

### Βήματα για προσαρμογή της αναλογίας διαστάσεων
1. **Δημιουργήστε μια παρουσία `BarcodeGenerator`** με `EncodeTypes.Aztec`.  
2. **Αναθέστε τα δεδομένα σας** (κείμενο, bytes ή αριθμητική συμβολοσειρά).  
3. **Ορίστε `AspectRatio`** – για παράδειγμα, `1.5` για πιο πλατύ μπαρ.  
4. **Δημιουργήστε την εικόνα** χρησιμοποιώντας `Save` ή `GetBarCodeImage`.  

## Πώς μπορείτε να κωδικοποιήσετε ακατέργαστα bytes σε έναν γραμμωτό κώδικα Aztec;
`EncodeBytes` είναι μια μέθοδος που δέχεται έναν πίνακα byte και τον μετατρέπει σε μια μήτρα Aztec. Περνάτε έναν πίνακα byte στη μέθοδο `EncodeBytes` του `BarcodeGenerator`. Το API μετατρέπει αυτόματα τα δυαδικά δεδομένα σε μια συμπαγή μήτρα Aztec, διατηρώντας κάθε bit. Αυτό είναι ιδανικό για ενσωμάτωση κρυπτογραφημένων φορτίων, δυαδικών αναγνωριστικών ή συμπιεσμένων αρχείων απευθείας σε έναν γραμμωτό κώδικα.

### Βήματα για κωδικοποίηση bytes
1. **Προετοιμάστε τον πίνακα byte** (π.χ., `byte[] data = Encoding.UTF8.GetBytes("Hello")`).  
2. **Δημιουργήστε μια παρουσία `BarcodeGenerator`** με `EncodeTypes.Aztec`.  
3. **Καλέστε `EncodeBytes(data)`** για να φορτώσετε το δυαδικό περιεχόμενο.  
4. **Αποδώστε τον γραμμωτό κώδικα** με `Save` ή `GetBarCodeImage`.  

## Πώς κωδικοποιείτε κείμενο σε έναν γραμμωτό κώδικα Aztec;
`CodeText` είναι μια ιδιότητα που περιέχει τα δεδομένα απλού κειμένου προς κωδικοποίηση. Χρησιμοποιήστε την ιδιότητα `CodeText` του `BarcodeGenerator` για να παρέχετε δεδομένα απλού κειμένου. Η βιβλιοθήκη επιλέγει αυτόματα τη βέλτιστη λειτουργία κωδικοποίησης (αριθμητική, αλφαριθμητική ή byte) και εφαρμόζει το κατάλληλο επίπεδο διόρθωσης σφαλμάτων. Αυτό καθιστά εύκολο το ενσωμάτωση URLs, IDs προϊόντων ή οποιασδήποτε αναγνώσιμης συμβολοσειράς.

### Βήματα για κωδικοποίηση κειμένου
1. **Δημιουργήστε τον δημιουργό** με `EncodeTypes.Aztec`.  
2. **Ορίστε `CodeText`** στη συμβολοσειρά που θέλετε να κωδικοποιήσετε.  
3. **Προαιρετικά προσαρμόστε το `ErrorLevel`** για μεγαλύτερη ανθεκτικότητα.  
4. **Δημιουργήστε την εικόνα** χρησιμοποιώντας `Save` ή `GetBarCodeImage`.  

## Πώς μπορείτε να δημιουργήσετε γραμμωτούς κώδικες Aztec με διαφορετικά επίπεδα διόρθωσης σφαλμάτων;
`ErrorLevel` είναι μια ιδιότητα που ελέγχει την ποσότητα των πλεονασματικών δεδομένων που προστίθενται στον γραμμωτό κώδικα. Προσαρμόστε την ιδιότητα `ErrorLevel` (0‑4) πριν από την απόδοση. Τα υψηλότερα επίπεδα αυξάνουν την ποσότητα των πλεονασματικών δεδομένων, επιτρέποντας την ανάγνωση του κώδικα ακόμη και όταν έως 30 % του συμβόλου είναι κατεστραμμένο. Αυτό είναι κρίσιμο για σκληρά περιβάλλοντα όπως η βιομηχανική σήμανση ή η εξωτερική διαφήμιση.

### Βήματα για ορισμό διόρθωσης σφαλμάτων
1. **Δημιουργήστε μια παρουσία `BarcodeGenerator`** για Aztec.  
2. **Αναθέστε τα δεδομένα σας** (κείμενο ή bytes).  
3. **Ορίστε `ErrorLevel`** – π.χ., `generator.Parameters.Barcode.Aztec.ErrorLevel = 3`.  
4. **Αποδώστε τον γραμμωτό κώδικα** με την προτιμώμενη μορφή εξόδου.  

## Ποιοι είναι οι διαφορετικοί τρόποι Συμβόλου Aztec και πώς τους χρησιμοποιείτε;
`SymbolMode` είναι μια ρύθμιση που καθορίζει το μέγεθος της μήτρας και τη χωρητικότητα δεδομένων του παραγόμενου κώδικα Aztec. Η λειτουργία Συμβόλου Aztec καθορίζει το μέγεθος της μήτρας και τη χωρητικότητα δεδομένων. Η βιβλιοθήκη προσφέρει τέσσερις λειτουργίες: **Auto**, **FullRange**, **Compact**, και **Rune**.

- **Auto** – ο δημιουργός επιλέγει το μικρότερο δυνατό μέγεθος.  
- **FullRange** – επιτρέπει το μέγιστο μέγεθος μήτρας για πολύ μεγάλα σύνολα δεδομένων.  
- **Compact** – δημιουργεί ένα μικρότερο, πιο πυκνό σύμβολο ιδανικό για περιορισμένο χώρο.  
- **Rune** – μια εξειδικευμένη λειτουργία για κωδικοποίηση Unicode runes.

Επιλέξτε τη λειτουργία μέσω `Generator.Parameters.Barcode.Aztec.SymbolMode`. Κάθε λειτουργία ισορροπεί το μέγεθος, την αναγνωσιμότητα και τη χωρητικότητα δεδομένων, επιτρέποντάς σας να προσαρμόσετε τον γραμμωτό κώδικα στις περιοριστικές απαιτήσεις της εφαρμογής σας.
