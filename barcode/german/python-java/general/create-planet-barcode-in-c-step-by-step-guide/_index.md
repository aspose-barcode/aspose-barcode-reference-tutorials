# Planet‑Barcode in C# erstellen – Schritt‑für‑Schritt‑Anleitung

Haben Sie jemals einen **Planet‑Barcode** erstellen müssen, wussten aber nicht, wo Sie anfangen sollen? In diesem Tutorial sehen Sie genau **wie man einen Planet‑Barcode generiert** mit der Aspose.Barcode‑Bibliothek, sowohl mit Standard‑ als auch mit benutzerdefinierten Höhenoptionen.

Wir führen Sie durch den gesamten Prozess – von der Installation des NuGet‑Pakets über das Anpassen der Balkenhöhe bis hin zum Speichern der PNG‑Dateien. Am Ende haben Sie vier einsatzbereite Bilder: zwei Planet‑Barcodes (automatisch und 100 Pixel Höhe) und zwei RM4SCC‑Barcodes zum Vergleich. Kein Schnickschnack, nur praktischer Code, den Sie in jedes .NET‑Projekt übernehmen können.

## Voraussetzungen

Bevor wir starten, stellen Sie sicher, dass Sie Folgendes haben:

* .NET 6.0 oder neuer (der Code funktioniert auch mit .NET Framework 4.7+)
* Visual Studio 2022 (oder eine IDE Ihrer Wahl)
* Das **Aspose.Barcode for .NET** NuGet‑Paket  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Einen Ordner auf dem Datenträger, in den die erzeugten PNG‑Dateien geschrieben werden können (ersetzen Sie `YOUR_DIRECTORY` im Code).

Falls Ihnen etwas davon unbekannt ist, keine Sorge – ein NuGet‑Paket zu installieren ist so einfach wie: Rechts‑Klick auf das Projekt → *Manage NuGet Packages* → nach „Aspose.Barcode“ suchen → *Install*.

## Schritt 1 – Minimalprojekt für die Konsole erstellen

Zuerst legen wir ein kleines Konsolen‑App‑Gerüst an, damit wir den Barcode‑Code isoliert ausführen können.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Pro‑Tipp:** Halten Sie die `Main`‑Methode übersichtlich; delegieren Sie die eigentliche Arbeit an separate Methoden. Das macht den Code leichter test‑ und lesbar.

## Schritt 2 – **Planet‑Barcode** mit Standardhöhe erstellen

Jetzt erzeugen wir tatsächlich einen **Planet‑Barcode** mit der standardmäßig (automatisch) berechneten Höhe. `XDimension` steuert die Breite eines einzelnen Moduls, während die Balkenhöhe der Bibliothek überlassen wird.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Warum funktioniert das?*  
`EncodeTypes.Planet` teilt Aspose mit, dass wir die Planet‑Symbologie wollen, und `"123456"` ist die zu codierende Daten­payload. Da `BarHeight` nicht gesetzt wird, wählt die Engine automatisch eine Höhe, die den Vorgaben entspricht.

### Erwartete Ausgabe

Öffnen Sie `PlanetBarHeightAuto.png` – Sie sollten einen sauberen Planet‑Barcode sehen, etwa 30 mm hoch (die genaue Größe hängt vom DPI‑Wert des PNG‑Writers ab). Keine manuelle Höhenanpassung nötig.

## Schritt 3 – **Planet‑Barcode** mit benutzerdefinierter 100‑Pixel‑Höhe erzeugen

Falls Ihr Design eine feste Höhe verlangt – zum Beispiel 100 Pixel – setzen Sie einfach die Eigenschaft `BarHeight`, bevor Sie speichern.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Beachten Sie die zusätzliche Zeile `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Damit wird der Barcode gezwungen, exakt 100 Pixel hoch zu sein, unabhängig von der Datenlänge.

> **Achtung:** Eine zu kleine Höhe kann dazu führen, dass die Balken zusammenlaufen und der Barcode unlesbar wird. Als Faustregel gilt: Die Höhe sollte mindestens das Dreifache der X‑Dimension betragen.

## Schritt 4 – RM4SCC‑Barcodes zum Vergleich erzeugen

Manchmal muss man Planet mit anderen Post‑Symbologien vergleichen. RM4SCC ist ein weiteres gängiges Format; der nachfolgende Code spiegelt die Schritte wider, die wir für Planet verwendet haben.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Führt man das komplette Programm jetzt aus, entstehen **vier** PNG‑Dateien:

| Datei | Symbologie | Höhe |
|------|------------|------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Sie können jede Bilddatei in einem beliebigen Viewer öffnen, um Größe und visuelle Qualität zu prüfen.

## Schritt 5 – Barcodes programmgesteuert verifizieren (optional)

Wenn Sie ganz sicher gehen wollen, dass die erzeugten Barcodes lesbar sind, bietet Aspose auch eine **Reader**‑Komponente. Hier ein kurzer Plausibilitäts‑Check, ob die codierten Daten dem ursprünglichen String entsprechen.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Beim Ausführen von `VerifyBarcodes()` wird `Decoded value: 123456` ausgegeben, was bestätigt, dass die benutzerdefinierte Höhe die Codierung nicht beeinträchtigt hat.

## Häufige Stolperfallen & Tipps

| Problem | Warum es passiert | Lösung |
|---------|-------------------|--------|
| Barcode erscheint zu dünn | X‑Dimension zu niedrig eingestellt (z. B. 1 Pixel) | `XDimension.Pixels` auf 3‑5 erhöhen für die meisten Drucker |
| Bild ist unscharf | Speicherung als PNG mit Standard‑DPI (96) auf hochauflösenden Bildschirmen | `ImageOptions` nutzen, um `ResolutionX` und `ResolutionY` auf 300 DPI zu setzen |
| Höhe wird nicht angewendet | `BarHeight` *nach* dem Aufruf von `Save()` gesetzt | `BarHeight` immer **vor** `Save()` setzen |
| Ungültiger Dateipfad | `YOUR_DIRECTORY` existiert nicht | Ordner programmatisch anlegen: `Directory.CreateDirectory("YOUR_DIRECTORY");` |

Das vollständige, ausführbare Beispiel befindet sich in einer einzigen `Program.cs`‑Datei, sodass Sie es in jedes .NET‑Projekt kopieren und sofort Barcodes drucken können.
