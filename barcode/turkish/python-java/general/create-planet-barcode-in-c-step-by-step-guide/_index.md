# C#'ta Planet Barkod Oluşturma – Adım Adım Kılavuz

Hiç **planet barkod oluşturmanız** gerekti ama nereden başlayacağınızı bilemediniz mi? Bu öğreticide Aspose.Barcode kütüphanesini kullanarak **planet barkod nasıl oluşturulur** gösterilecek ve hem varsayılan hem de özel yükseklik seçenekleri ele alınacak.

Tüm süreci adım adım inceleyeceğiz—NuGet paketini kurmaktan bar‑yüksekliğini ayarlamaya ve PNG dosyalarını kaydetmeye kadar. Sonunda dört kullanıma hazır görüntünüz olacak: iki Planet barkod (otomatik ve 100 piksel yüksekliğinde) ve iki RM4SCC barkod karşılaştırması için. Gereksiz ayrıntı yok, sadece herhangi bir .NET projesine ekleyebileceğiniz pratik kod.

## Önkoşullar

* .NET 6.0 veya üzeri (kod .NET Framework 4.7+ üzerinde de çalışır)
* Visual Studio 2022 (veya tercih ettiğiniz herhangi bir IDE)
* **Aspose.Barcode for .NET** NuGet paketi  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Üretilen PNG dosyalarının yazılabileceği bir klasör (kod içinde `YOUR_DIRECTORY` ifadesini değiştirin).

Eğer bu konular size yabancı geliyorsa endişelenmeyin—NuGet paketi kurmak proje üzerine sağ‑tıklayıp → *Manage NuGet Packages* → “Aspose.Barcode” aratıp → *Install* seçmek kadar kolaydır.

## Adım 1 – Minimal Bir Konsol Projesi Oluşturma

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Pro tip:** `Main` metodunu sade tutun; ağır işleri ayrı metodlara devredin. Böylece kodu daha sonra test etmek ve okumak kolaylaşır.

## Adım 2 – **Planet Barkod Oluştur** Varsayılan Yükseklik ile

Şimdi **planet barkod oluştur** ve varsayılan (otomatik‑hesaplanan) yüksekliği kullan. `XDimension` tek bir modülün genişliğini kontrol eder, bar yüksekliği ise kütüphaneye bırakılır.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Why does this work?*  
`EncodeTypes.Planet` Aspose’a Planet sembolünü istediğimizi söyler ve `"123456"` veri yüküdür. `BarHeight` ayarlanmadan, motor spesifikasyonu otomatik olarak karşılayan bir yükseklik seçer.

### Beklenen Çıktı

`PlanetBarHeightAuto.png` dosyasını açtığınızda temiz bir Planet barkod görmelisiniz; yaklaşık 30 mm yüksekliğinde (tam boyut PNG yazarının DPI değerine bağlıdır). Manuel yükseklik ayarı gerekmez.

## Adım 3 – **Planet Barkod Nasıl Oluşturulur** Özel 100 Piksel Yükseklik ile

Tasarımınız sabit bir yükseklik gerektiriyorsa—örneğin 100 piksel—kaydetmeden önce `BarHeight` özelliğini ayarlamanız yeterlidir.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

`planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;` satırına dikkat edin. Bu satır, veri uzunluğundan bağımsız olarak barkodu tam 100 piksel yüksekliğe zorlar.

> **Watch out:** Çok düşük bir yükseklik ayarlamak, çubukların birleşmesine ve barkodun okunamaz hale gelmesine neden olabilir. Güvenli bir kural, yüksekliği X‑dimension’ın en az üç katı tutmaktır.

## Adım 4 – Karşılaştırma İçin RM4SCC Barkodları Oluşturma

Bazen Planet’i diğer posta sembolleriyle karşılaştırmanız gerekir. RM4SCC başka yaygın bir formattır; aşağıdaki kod Planet için izlediğimiz adımları yansıtır.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Tam programı çalıştırdığınızda **dört** PNG dosyası üretilecektir:

| Dosya | Semboloji | Yükseklik |
|------|-----------|-----------|
| `PlanetBarHeightAuto.png` | Planet | Otomatik |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Otomatik |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Her bir görüntüyü herhangi bir görüntüleyicide açarak boyutları ve görsel kaliteyi doğrulayabilirsiniz.

## Adım 5 – Barkodları Programatik Olarak Doğrulama (İsteğe Bağlı)

Oluşturulan barkodların taranabilir olduğundan ekstra emin olmak isterseniz, Aspose ayrıca bir **reader** bileşeni sunar. İşte kodlanmış verinin orijinal dizeyle eşleştiğini hızlıca kontrol eden bir test.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

`VerifyBarcodes()` çalıştırıldığında `Decoded value: 123456` çıktısını verir ve özel yüksekliğin kodlamayı bozmadığını onaylar.

## Yaygın Tuzaklar ve İpuçları

| Sorun | Neden Oluşur | Çözüm |
|-------|--------------|-------|
| Barkod çok ince görünüyor | X‑dimension çok düşük ayarlanmış (ör. 1 piksel) | Çoğu yazıcı için `XDimension.Pixels` değerini 3‑5 arasına yükseltin |
| Görüntü bulanık | Varsayılan DPI (96) ile PNG kaydedildiğinde yüksek çözünürlüklü ekranlarda | `ImageOptions` ile `ResolutionX` ve `ResolutionY` değerlerini 300 DPI olarak ayarlayın |
| Yükseklik uygulanmadı | `BarHeight` **Save()** çağrısından **sonra** ayarlandı | `BarHeight` değerini **Save()**'dan önce ayarlayın |
| Dosya yolu geçersiz | `YOUR_DIRECTORY` mevcut değil | Klasörü programatik olarak oluşturun: `Directory.CreateDirectory("YOUR_DIRECTORY");` |

## Özet

Sıfırdan **planet barkod** görüntüleri **oluşturduk**, hem otomatik hem de belirli yüksekliklerle **planet barkod nasıl oluşturulur** konusunu inceledik ve bunları RM4SCC eşdeğerleriyle yan yana karşılaştırdık. Tam, çalıştırılabilir örnek tek bir `Program.cs` dosyasında yer alıyor; böylece kopyalayıp herhangi bir .NET projesine yapıştırarak barkodları hemen yazdırmaya başlayabilirsiniz.

### Sıradaki Adımlar

* Farklı `XDimension` değerleri deneyerek modül genişliğinin tarama güvenilirliğini nasıl etkilediğini görün.  
* `ImageOptions.ForeColor` ve `BackColor` ayarlarıyla renkli çıktılar deneyin.  
* Aynı desenle diğer posta sembollerine (ör. **Postnet**, **Intelligent Mail**) dalın.
