# Generowanie kodu kreskowego Code128 przy użyciu Aspose.Barcode Python – Pełny przewodnik

Czy kiedykolwiek potrzebowałeś **wygenerować kod kreskowy Code128** w projekcie Python, ale nie wiedziałeś, jak osadzić znaki nie‑ASCII? Nie jesteś sam. Wielu programistów napotyka problem, gdy próbują połączyć dane liczbowe z tekstem Unicode — szczególnie rosyjskim lub chińskim — w jednym kodzie kreskowym. Dobra wiadomość? Aspose.Barcode for Python ułatwia to, a przy odpowiednim użyciu *barcode with ECI* możesz zapewnić czytelność dla skanerów na całym świecie.

W tym samouczku przeprowadzimy Cię przez kompletny, działający przykład, który dokładnie pokazuje, jak **wygenerować kod kreskowy Code128** przy użyciu biblioteki Aspose.Barcode Python, jak stworzyć rozszerzony ciąg kodowy zawierający segment ECI oraz dlaczego każda linia ma znaczenie. Po zakończeniu będziesz mieć gotowy do uruchomienia skrypt, jasne zrozumienie podstawowych koncepcji oraz kilka profesjonalnych wskazówek, jak uniknąć typowych pułapek.

## Czego się nauczysz

- Konfiguracja **Aspose.Barcode Python** w Twoim środowisku.  
- Tworzenie rozszerzonego ciągu kodowego, który miesza zwykłe liczby z tekstem UTF‑8 za pomocą segmentu ECI.  
- Konfigurowanie `BarCodeGenerator` do generowania **kod kreskowy Code128**.  
- Zapisywanie obrazu i weryfikacja wyniku.  
- Obsługa przypadków brzegowych dla różnych zestawów znaków i symboli kodów kreskowych.

Nie potrzebna jest żadna zewnętrzna dokumentacja — wszystko, czego potrzebujesz, znajduje się tutaj.

---

## Wymagania wstępne

Zanim zaczniemy, upewnij się, że masz:

1. **Python 3.8+** zainstalowany (biblioteka obsługuje 3.6 i nowsze).  
2. Ważną licencję **Aspose.Barcode for Python via .NET** lub akceptujesz znak wodny wersji ewaluacyjnej.  
3. Dostęp do terminala lub IDE, w którym możesz instalować pakiety (`pip`).  
4. Folder, w którym zostanie zapisany wygenerowany PNG (zamień `YOUR_DIRECTORY` w kodzie).

Jeśli któreś z tych zagadnień jest Ci nieznane, nie martw się — instalacja pakietu to jedno polecenie, a reszta to czysty Python.

## Krok 1: Zainstaluj Aspose.Barcode dla Pythona

Otwórz terminal i uruchom:

```bash
pip install aspose-barcode
```

> **Wskazówka:** Użyj wirtualnego środowiska (`python -m venv venv`), aby izolować zależności. To także zapobiega konfliktom wersji z innymi projektami.

Pakiet w tle pobiera środowisko .NET, więc pierwsze importowanie może chwilę potrwać, gdy ładowane są natywne biblioteki.

## Krok 2: Utwórz konstruktor rozszerzonego ciągu kodowego

Serce wsparcia **barcode with ECI** znajduje się w `ExtCodetextBuilder`. Umożliwia on łączenie zwykłych segmentów z częściami zakodowanymi ECI, zapewniając prawidłową interpretację tekstu Unicode przez skanery.

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**Dlaczego wyczyścić?**  
Jeśli ponownie używasz buildera w pętli, pozostające dane mogą zepsuć kolejny kod kreskowy. Wywołanie `clear()` zapewnia czystą kartkę za każdym razem.

## Krok 3: Dodaj zwykły segment liczbowy

Większość kodów kreskowych Code128 zaczyna się od identyfikatora numerycznego — np. numeru zamówienia lub ID produktu. Dodanie go jako segmentu *plain* utrzymuje kodowanie proste i zwarte.

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

Metoda automatycznie wybiera optymalny podzbiór Code128 (A, B lub C) dla cyfr, co skutkuje najmniejszym możliwym kodem kreskowym.

## Krok 4: Wstaw segment ECI z rosyjskim tekstem

Teraz trudna część: osadzenie rosyjskich znaków (`Пример текста`). Bez znacznika ECI większość skanerów błędnie zinterpretowałaby te bajty, generując nieczytelny wynik.

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**Co się dzieje?**  
- `26` informuje kod kreskowy, że kolejne bajty są w kodowaniu UTF‑8.  
- Biblioteka zajmuje się konwersją, więc po prostu przekazujesz ciąg Unicode.  
- Gdy skaner odczytuje kod, widzi ECI, przełącza się na UTF‑8 i prawidłowo wyświetla rosyjski tekst.

> **Częste pytanie:** *Co jeśli potrzebuję innego języka?*  
> Użyj odpowiedniej wartości ECI (np. 27 dla ISO‑8859‑1, 28 dla ISO‑8859‑2). Szybka tabela dostępna jest w dokumentacji Aspose, ale wzorzec pozostaje taki sam.

## Krok 5: Pobierz pełny rozszerzony ciąg kodowy

Po dodaniu obu segmentów pobieramy połączony ciąg. To jest wartość, którą przypiszesz do `code_text` generatora.

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

Wydrukowany wynik wygląda mniej więcej tak:

```
Extended codetext: 1234567890[ECI26]Пример текста
```

Token `[ECI26]` jest niewidoczny w ostatecznym obrazie, ale kluczowy dla prawidłowego dekodowania.

## Krok 6: Skonfiguruj BarCodeGenerator dla Code128

Teraz łączymy wszystko. `BarCodeGenerator` to silnik, który renderuje obraz.

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**Dlaczego ustawia się `x_dimension`?**  
Mniejsza szerokość modułu daje bardziej zwarty kod kreskowy — przydatne przy ograniczonej przestrzeni. Dostosuj ją do DPI Twojej drukarki.

## Krok 7: Zapisz obraz kodu kreskowego

Na koniec zapisz PNG na dysku. Możesz także wyeksportować do JPEG, BMP lub nawet SVG, zmieniając rozszerzenie pliku.

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

Otwórz obraz w dowolnym przeglądarce; powinieneś zobaczyć czysty kod kreskowy Code128, który po zeskanowaniu nowoczesnym czytnikiem zwróci `1234567890Пример текста`.

## Pełny skrypt – wklej i uruchom

Poniżej znajduje się kompletny, samodzielny skrypt. Skopiuj go do pliku o nazwie `generate_code128.py`, dostosuj `YOUR_DIRECTORY` i uruchom `python generate_code128.py`.

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**Oczekiwany wynik** (konsola):

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

Otwórz `extended_code.png` i zobaczysz wyraźny kod kreskowy Code128 gotowy do użycia w produkcji.

## Szczegółowo: Jak działa mechanizm ECI

ECI (Extended Channel Interpretation) jest częścią standardów QR i kodów kreskowych, które sygnalizują zmianę kodowania znaków w trakcie transmisji. W kontekście Code128:

1. Kod kreskowy zaczyna w domyślnym trybie ISO‑8859‑1.  
2. Gdy skaner napotka kod kontrolny ECI (wartość 26 w naszym przykładzie), przełącza się na UTF‑8.  
3. Wszystkie kolejne bajty są interpretowane jako UTF‑8 aż do kolejnego ECI lub końca danych.

Ponieważ wyraźnie ustawiliśmy `add_eci_codetext(26, ...)`, Aspose wstawia odpowiednie znaki kontrolne w tle. Jeśli pominiesz ECI, rosyjski tekst zostanie zakodowany przy użyciu domyślnej strony kodowej, co zazwyczaj skutkuje nieczytelnym zniekształceniem na większości skanerów.

## Obsługa przypadków brzegowych

### 1. Wiele segmentów ECI

Jeśli musisz osadzić zarówno japoński, jak i arabski tekst, możesz wywołać `add_eci_codetext` wielokrotnie z różnymi wartościami ECI. Builder łączy je w kolejności, w jakiej je dodajesz.

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. Limity długości

Code128 może zakodować do 2 023 znaków, ale praktyczny limit jest często niższy ze względu na pamięć skanera. Jeśli go przekroczysz, generator zgłosi `ArgumentException`. Aby być bezpiecznym, utrzymuj łączną długość poniżej 1 000 znaków przy mieszaniu segmentów ECI.

### 3. Kompatybilność ze starszymi skanerami

Niektóre starsze skanery ignorują kody kontrolne ECI. W takich środowiskach może być konieczne generowanie oddzielnych kodów kreskowych — jednego dla części numerycznej, drugiego dla tekstu Unicode — lub użycie innej symboliki, takiej jak QR, która ma szersze wsparcie ECI.

## Profesjonalne wskazówki dla zastosowań produkcyjnych

- **Licencja od razu:** Wersja ewaluacyjna dodaje mały znak wodny. Zastosuj plik licencyjny przed pierwszym wywołaniem `BarCodeGenerator`, aby go uniknąć.  
- **Przetwarzanie wsadowe:** Przy generowaniu tysięcy kodów kreskowych, ponownie używaj jednej instancji `ExtCodetextBuilder` — po prostu wywołuj `clear()` w każdej iteracji pętli.  
- **Bezpieczeństwo wątków:** Każdy wątek powinien posiadać własną instancję `BarCodeGenerator`; obiekty nie są bezpieczne wątkowo.  
- **Wydajność:** Ustawienie `x_dimension` zbyt niskiego może wydłużyć czas renderowania. Przeprowadź benchmark na docelowym sprzęcie, jeśli pracujesz w scenariuszu wysokiej przepustowości.  
- **Testowanie:** Zawsze skanuj wygenerowany kod kreskowy przynajmniej dwoma różnymi czytnikami (aplikacja mobilna i skaner sprzętowy), aby zweryfikować obsługę ECI.
