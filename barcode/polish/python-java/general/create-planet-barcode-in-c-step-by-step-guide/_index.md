# Utwórz kod kreskowy Planet w C# – Przewodnik krok po kroku

Kiedykolwiek potrzebowałeś **utworzyć kod kreskowy Planet**, ale nie wiedziałeś, od czego zacząć? W tym samouczku zobaczysz dokładnie **jak wygenerować kod kreskowy Planet** przy użyciu biblioteki Aspose.Barcode, zarówno z domyślną, jak i niestandardową wysokością.

Przejdziemy przez cały proces — od instalacji pakietu NuGet po dostosowanie wysokości pasków i zapisanie plików PNG. Na końcu będziesz mieć cztery gotowe obrazy: dwa kody Planet (automatyczna i wysokość 100 pikseli) oraz dwa kody RM4SCC do porównania. Bez zbędnych wstępów, tylko praktyczny kod, który możesz wkleić do dowolnego projektu .NET.

## Wymagania wstępne

Przed rozpoczęciem upewnij się, że masz:

* .NET 6.0 lub nowszy (kod działa również na .NET Framework 4.7+)
* Visual Studio 2022 (lub dowolne IDE, które preferujesz)
* Pakiet NuGet **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Folder na dysku, w którym można zapisać wygenerowane pliki PNG (zastąp `YOUR_DIRECTORY` w kodzie).

Jeśli którykolwiek z tych elementów jest Ci nieznany, nie martw się — instalacja pakietu NuGet jest tak prosta, jak kliknięcie prawym przyciskiem myszy na projekt → *Manage NuGet Packages* → wyszukanie „Aspose.Barcode” → *Install*.

## Krok 1 – Utwórz minimalny projekt konsolowy

Najpierw przygotujmy małą aplikację konsolową, aby móc uruchomić kod generujący kod kreskowy w izolacji.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Pro tip:** Utrzymuj metodę `Main` schludną; deleguj ciężkie operacje do osobnych metod. Dzięki temu kod jest łatwiejszy do testowania i późniejszego czytania.

## Krok 2 – **Utwórz kod kreskowy Planet** z domyślną wysokością

Teraz faktycznie **tworzymy kod kreskowy Planet** używając domyślnej (automatycznie obliczanej) wysokości. `XDimension` kontroluje szerokość pojedynczego modułu, a wysokość pasków pozostaje w gestii biblioteki.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Dlaczego to działa?*  
`EncodeTypes.Planet` informuje Aspose, że chcemy symbologię Planet, a `"123456"` jest ładunkiem danych. Nie ustawiając `BarHeight`, silnik automatycznie wybiera wysokość spełniającą specyfikację.

### Oczekiwany wynik

Otwórz `PlanetBarHeightAuto.png` i powinieneś zobaczyć czysty kod Planet, mający około 30 mm wysokości (dokładny rozmiar zależy od DPI zapisu PNG). Nie wymaga ręcznego dopasowywania wysokości.

## Krok 3 – **Jak wygenerować kod kreskowy Planet** z niestandardową wysokością 100 pikseli

Jeśli Twój projekt wymaga stałej wysokości — powiedzmy 100 pikseli — po prostu ustaw właściwość `BarHeight` przed zapisem.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Zauważ dodatkową linię `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Wymusza ona rozciągnięcie (lub skurczenie) kodu do dokładnie 100 pikseli wysokości, niezależnie od długości danych.

> **Uwaga:** Ustawienie zbyt małej wysokości może spowodować połączenie pasków, co uczyni kod nieczytelnym. Bezpieczną zasadą jest utrzymanie wysokości co najmniej trzykrotności X‑dimension.

## Krok 4 – Generowanie kodów RM4SCC do porównania

Czasami trzeba porównać Planet z innymi symbologiami pocztowymi. RM4SCC to kolejny popularny format; poniższy kod odzwierciedla kroki, które zastosowaliśmy dla Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Uruchomienie pełnego programu generuje **cztery** pliki PNG:

| Plik | Symbolika | Wysokość |
|------|-----------|----------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Możesz otworzyć każdy obraz w dowolnym przeglądarce, aby potwierdzić wymiary i jakość wizualną.

## Krok 5 – Weryfikacja kodów kreskowych programowo (opcjonalnie)

Jeśli chcesz mieć pewność, że wygenerowane kody są skanowalne, Aspose oferuje także komponent **reader**. Oto szybka kontrola, czy zakodowane dane zgadzają się z oryginalnym ciągiem.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Uruchomienie `VerifyBarcodes()` wypisuje `Decoded value: 123456`, potwierdzając, że niestandardowa wysokość nie zepsuła kodowania.

## Typowe problemy i wskazówki

| Problem | Dlaczego się pojawia | Rozwiązanie |
|---------|----------------------|-------------|
| Kod kreskowy wydaje się zbyt cienki | X‑dimension ustawione zbyt nisko (np. 1 piksel) | Zwiększ `XDimension.Pixels` do 3‑5 dla większości drukarek |
| Obraz jest rozmyty | Zapis jako PNG z domyślnym DPI (96) na ekranach wysokiej rozdzielczości | Użyj `ImageOptions`, aby ustawić `ResolutionX` i `ResolutionY` na 300 DPI |
| Wysokość nie została zastosowana | `BarHeight` ustawione *po* wywołaniu `Save()` | Zawsze ustaw `BarHeight` **przed** `Save()` |
| Ścieżka pliku nieprawidłowa | `YOUR_DIRECTORY` nie istnieje | Utwórz folder programowo: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
