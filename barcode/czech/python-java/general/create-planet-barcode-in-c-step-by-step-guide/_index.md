# Vytvoření Planet čárového kódu v C# – krok za krokem průvodce

Už jste někdy potřebovali **vytvořit planet čárový kód**, ale nebyli jste si jisti, kde začít? V tomto tutoriálu uvidíte přesně **jak generovat planet čárový kód** pomocí knihovny Aspose.Barcode, s možnostmi výchozí i vlastní výšky.

Provedeme vás celým procesem — od instalace NuGet balíčku po úpravu výšky čáry a uložení PNG souborů. Na konci budete mít čtyři připravené obrázky: dva Planet čárové kódy (automatická a výška 100 pixelů) a dva RM4SCC čárové kódy pro srovnání. Žádné zbytečnosti, jen praktický kód, který můžete vložit do libovolného .NET projektu.

## Požadavky

* .NET 6.0 nebo novější (kód funguje také na .NET Framework 4.7+)
* Visual Studio 2022 (nebo jakékoli IDE, které preferujete)
* Balíček **Aspose.Barcode for .NET** NuGet  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Složka na disku, kam mohou být zapisovány vygenerované PNG soubory (nahraďte `YOUR_DIRECTORY` v kódu).

Pokud vám některá z položek není známá, nebojte se — instalace NuGet balíčku je tak jednoduchá jako kliknout pravým tlačítkem na projekt → *Manage NuGet Packages* → vyhledat “Aspose.Barcode” → *Install*.

## Krok 1 — Vytvoření minimálního konzolového projektu

Nejprve vytvoříme malou konzolovou aplikaci, abychom mohli spustit kód čárového kódu izolovaně.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Tip:** Udržujte metodu `Main` přehlednou; delegujte těžkou práci do samostatných metod. Usnadní to testování a pozdější čtení kódu.

## Krok 2 — **Vytvoření Planet čárového kódu** s výchozí výškou

Nyní skutečně **vytvoříme planet čárový kód** pomocí výchozí (automaticky vypočítané) výšky. `XDimension` určuje šířku jednoho modulu, zatímco výška čáry je ponechána knihovně.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Proč to funguje?*  
`EncodeTypes.Planet` říká Aspose, že chceme symbologii Planet, a `"123456"` je datová část. Pokud nenastavíte `BarHeight`, engine automaticky vybere výšku, která splňuje specifikaci.

### Očekávaný výstup

Otevřete `PlanetBarHeightAuto.png` a měli byste vidět čistý Planet čárový kód, přibližně 30 mm vysoký (přesná velikost závisí na DPI PNG zapisovače). Není potřeba ručně upravovat výšku.

## Krok 3 — **Jak generovat Planet čárový kód** s vlastní výškou 100 pixelů

Pokud váš návrh vyžaduje pevnou výšku — například 100 pixelů — stačí nastavit vlastnost `BarHeight` před uložením.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Všimněte si extra řádku `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Ten vynutí, aby se čárový kód roztáhl (nebo zmenšil) přesně na 100 pixelů výšky, bez ohledu na délku dat.

> **Pozor:** Nastavení výšky, která je příliš malá, může způsobit sloučení čar a nečitelnost čárového kódu. Bezpečné pravidlo je udržet výšku alespoň třikrát větší než X‑dimenzia.

## Krok 4 — Generování RM4SCC čárových kódů pro srovnání

Někdy potřebujete porovnat Planet s jinými poštovními symbologiemi. RM4SCC je další běžný formát; níže uvedený kód odráží kroky, které jsme použili pro Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Spuštěním kompletního programu nyní získáte **čtyři** PNG soubory:

| Soubor | Symbologie | Výška |
|--------|------------|-------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Můžete otevřít každý obrázek v libovolném prohlížeči a ověřit rozměry a vizuální kvalitu.

## Krok 5 — Programová verifikace čárových kódů (volitelné)

Pokud chcete mít naprostou jistotu, že vygenerované čárové kódy jsou čitelné, Aspose také nabízí komponentu **reader**. Zde je rychlá kontrola, že zakódovaná data odpovídají našemu původnímu řetězci.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Spuštěním `VerifyBarcodes()` se vypíše `Decoded value: 123456`, což potvrzuje, že vlastní výška neporušila kódování.

## Časté úskalí a tipy

| Problém | Proč se to děje | Řešení |
|---------|----------------|--------|
| Čárový kód se zdá příliš tenký | X‑dimenze nastavena příliš nízko (např. 1 pixel) | Zvyšte `XDimension.Pixels` na 3‑5 pro většinu tiskáren |
| Obrázek je rozmazaný | Ukládání jako PNG s výchozím DPI (96) na obrazovkách s vysokým rozlišením | Použijte `ImageOptions` k nastavení `ResolutionX` a `ResolutionY` na 300 DPI |
| Výška nebyla použita | `BarHeight` nastaveno *po* volání `Save()` | Vždy nastavte `BarHeight` **před** `Save()` |
| Neplatná cesta k souboru | `YOUR_DIRECTORY` neexistuje | Vytvořte složku programově: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
