# 在 C# 中建立 Planet 條碼 – 步驟指南

是否曾需要 **建立 Planet 條碼**，卻不知從何入手？在本教學中，你將會看到如何使用 Aspose.Barcode 函式庫 **產生 Planet 條碼**，並提供預設與自訂高度兩種選項。

我們會一步步說明整個流程——從安裝 NuGet 套件、調整條碼高度到儲存 PNG 檔案。完成後，你將擁有四張可直接使用的圖像：兩張 Planet 條碼（自動高度與 100 像素高度）以及兩張 RM4SCC 條碼作比較。內容精簡實用，隨時可直接放入任何 .NET 專案中使用。

## 前置條件

* .NET 6.0 或更新版本（此程式碼亦可於 .NET Framework 4.7+ 執行）
* Visual Studio 2022（或任何你偏好的 IDE）
* Aspose.Barcode for .NET 的 **NuGet** 套件  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* 磁碟上可寫入產生 PNG 檔案的資料夾（請在程式碼中將 `YOUR_DIRECTORY` 替換為實際路徑）。

如果上述項目對你來說陌生，也別擔心——安裝 NuGet 套件非常簡單，只要在專案上點右鍵 → *Manage NuGet Packages* → 搜尋 “Aspose.Barcode” → *Install* 即可。

## 第一步 – 建立最小化的 Console 專案

首先，讓我們建立一個簡易的 console 應用程式，以便能單獨執行條碼程式碼。

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **小技巧：** 保持 `Main` 方法簡潔；將繁重的工作委派給其他方法。這樣可以讓程式碼更易於測試與閱讀。

## 第二步 – 使用預設高度 **建立 Planet 條碼**

現在我們實際上使用預設（自動計算）高度 **建立 Planet 條碼**。`XDimension` 控制單一模組的寬度，而條碼高度則交由函式庫自行決定。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*為什麼會這樣運作？*  
`EncodeTypes.Planet` 告訴 Aspose 我們要使用 Planet 符號，而 `"123456"` 為資料內容。若未設定 `BarHeight`，引擎會自動挑選符合規範的高度。

### 預期輸出

開啟 `PlanetBarHeightAuto.png`，你應該會看到一個清晰的 Planet 條碼，約 30 mm 高（實際尺寸取決於 PNG 寫入器的 DPI）。不需要手動調整高度。

## 第三步 – 使用自訂 100 像素高度 **產生 Planet 條碼**

如果你的設計需要固定高度——例如 100 像素——只要在儲存前設定 `BarHeight` 屬性即可。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

請注意額外的程式碼 `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`。此設定會強制條碼無論資料長度如何，都拉伸（或縮小）至恰好 100 像素高。

> **注意：** 設定過小的高度會導致條紋合併，使條碼無法辨識。安全的經驗法則是高度至少為 X‑dimension 的三倍。

## 第四步 – 產生 RM4SCC 條碼作比較

有時你需要將 Planet 與其他郵政條碼做比較。RM4SCC 是另一常見格式；以下程式碼與 Planet 的步驟相同。

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

執行完整程式後會產生 **四** 個 PNG 檔案：

| 檔案 | 符號 | 高度 |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

你可以使用任何影像檢視器開啟每張圖片，以確認尺寸與視覺品質。

## 第五步 – 以程式方式驗證條碼（可選）

如果想更確定產生的條碼可被掃描，Aspose 也提供 **reader** 元件。以下是一個快速的檢查，確認編碼資料與原始字串相符。

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

執行 `VerifyBarcodes()` 會輸出 `Decoded value: 123456`，證實自訂高度未影響編碼。

## 常見問題與技巧

| 問題 | 發生原因 | 解決方法 |
|-------|----------------|-----|
| 條碼顯得太細 | X‑dimension 設定過低（例如 1 像素） | 將 `XDimension.Pixels` 提升至 3‑5（適用於大多數印表機） |
| 影像模糊 | 在高解析度螢幕上以預設 DPI（96）儲存 PNG | 使用 `ImageOptions` 將 `ResolutionX` 與 `ResolutionY` 設為 300 DPI |
| 高度未套用 | `BarHeight` 在呼叫 `Save()` 之後才設定 | 務必在 `Save()` 之前設定 `BarHeight` |
| 檔案路徑無效 | `YOUR_DIRECTORY` 不存在 | 以程式方式建立資料夾：`Directory.CreateDirectory("YOUR_DIRECTORY");` |
