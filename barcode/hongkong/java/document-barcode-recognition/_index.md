## 為何在 Java 中使用 Aspose.BarCode 進行條碼偵測？

- **文件感知處理** — 可直接處理 PDF 與 Word 檔案，無需先轉換為影像。  
- **效能最佳化** — 支援批次處理與多執行緒。

## 常見問題

**Q: 可以在商業專案中使用 Aspose.BarCode 嗎？**  
A: 可以。正式環境需使用有效的 Aspose 授權，免費試用版可供評估使用。

**Q: 此函式庫能處理受密碼保護的 PDF 嗎？**  
A: 完全支援。載入文件時提供 PDF 密碼，即可正常執行條碼偵測。

**Q: 支援哪些 Java 版本？**  
A: API 支援 Java 8 及以上版本，包括 Java 11、Java 17 以及更新的 LTS 版本。

**Q: 如何提升大量文件批次的偵測速度？**  
A: 在多執行緒模式下使用 `BarCodeReader`，並啟用 `ReadQuality` 選項，以在速度與準確度之間取得平衡。

**Q: 若未偵測到條碼該怎麼辦？**  
A: 請確認條碼符號系統受支援，且影像解析度符合最低 DPI 建議（大多數 1D 條碼需 300 dpi）。
