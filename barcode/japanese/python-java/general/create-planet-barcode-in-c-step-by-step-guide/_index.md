# C# で Planet バーコードを作成 – ステップバイステップガイド

Planet バーコードを **作成** したいと思ったことはありますか？でもどこから始めればいいかわからない… このチュートリアルでは、Aspose.Barcode ライブラリを使用して **planet barcode を生成する方法** を、デフォルトとカスタムの高さオプションの両方で具体的に示します。

NuGet パッケージのインストールからバーの高さの調整、PNG ファイルの保存まで、全工程を順に解説します。最後には、使い ready な画像が 4 枚手に入ります：Planet バーコードが 2 枚（自動高さと 100 ピクセル高さ）と、比較用の RM4SCC バーコードが 2 枚です。余計な説明は省き、.NET プロジェクトにそのまま貼り付けられる実用的なコードだけを提供します。

## 前提条件

* .NET 6.0 以降（コードは .NET Framework 4.7+ でも動作します）
* Visual Studio 2022（またはお好みの IDE）
* The **Aspose.Barcode for .NET** NuGet package  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* 生成された PNG ファイルを書き込めるディスク上のフォルダー（コード内の `YOUR_DIRECTORY` を置き換えてください）。

これらに馴染みがなくても心配はいりません。NuGet パッケージのインストールは、プロジェクトを右クリック → *Manage NuGet Packages* → “Aspose.Barcode” を検索 → *Install* ととても簡単です。

## ステップ 1 – 最小限のコンソールプロジェクトを作成

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **プロのコツ:** `Main` メソッドはシンプルに保ち、重い処理は別メソッドに委譲しましょう。これにより、コードのテストや後からの読みやすさが向上します。

## ステップ 2 – デフォルト高さで **Planet バーコードを作成**

ここでは、デフォルト（自動計算）高さで **planet barcode を作成** します。`XDimension` は単一モジュールの幅を制御し、バーの高さはライブラリに任せます。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*なぜこれが機能するのか？*  
`EncodeTypes.Planet` は Aspose に Planet シンボルを使用したいことを指示し、`"123456"` がデータペイロードです。`BarHeight` を設定しないことで、エンジンは仕様を満たす高さを自動的に選択します。

### 期待される出力

`PlanetBarHeightAuto.png` を開くと、約 30 mm の高さのきれいな Planet バーコードが表示されます（正確なサイズは PNG ライターの DPI に依存します）。手動で高さを調整する必要はありません。

## ステップ 3 – カスタム 100 ピクセル高さで **Planet バーコードを生成する方法**

デザイン上、固定高さ（例：100 ピクセル）が必要な場合は、保存前に `BarHeight` プロパティを設定するだけです。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

余分な行 `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;` に注目してください。これにより、データ長に関係なくバーコードの高さが正確に 100 ピクセルになるように伸縮させます。

> **注意:** 高さを小さく設定しすぎるとバーが重なり合い、バーコードが読めなくなります。安全策として、高さは X‑dimension の少なくとも 3 倍に保つことを推奨します。

## ステップ 4 – 比較用に RM4SCC バーコードを生成

Planet と他の郵便シンボルを比較したいことがあります。RM4SCC は別の一般的なフォーマットで、以下のコードは Planet 用に行った手順と同様です。

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

プログラムを実行すると、**4** つの PNG ファイルが生成されます：

| ファイル | シンボロジー | 高さ |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | 自動 |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | 自動 |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

各画像を任意のビューアで開き、サイズと視覚的品質を確認できます。

## ステップ 5 – バーコードをプログラムで検証する（オプション）

生成されたバーコードが確実にスキャン可能か確認したい場合、Aspose には **リーダー** コンポーネントも用意されています。以下は、エンコードされたデータが元の文字列と一致するかを簡単にチェックする例です。

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

`VerifyBarcodes()` を実行すると `Decoded value: 123456` が出力され、カスタム高さがエンコードを壊していないことが確認できます。

## よくある落とし穴とヒント

| 問題点 | 発生原因 | 対策 |
|-------|----------------|-----|
| バーコードが細すぎる | X‑dimension が低すぎる（例：1 ピクセル） | `XDimension.Pixels` を 3‑5 に増やすと多くのプリンターで適切です |
| 画像がぼやける | 高解像度画面でデフォルト DPI（96）の PNG として保存している | `ImageOptions` で `ResolutionX` と `ResolutionY` を 300 DPI に設定する |
| 高さが適用されない | `BarHeight` を `Save()` 後に設定している | `BarHeight` は必ず `Save()` **前** に設定する |
| ファイルパスが無効 | `YOUR_DIRECTORY` が存在しない | フォルダーをプログラムで作成する: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
