# Python 用 Aspose.Barcode で Code128 バーコードを生成する – 完全ガイド

Python プロジェクトで **Code128 バーコード** を生成したいけれど、非 ASCII 文字を埋め込む方法が分からない…という経験はありませんか？同じ悩みを抱える開発者は多いです。数値データと Unicode テキスト（特にロシア語や中国語）を 1 つのバーコードに混在させようとすると壁にぶつかります。朗報です！Aspose.Barcode for Python を使えば簡単に実現でき、*barcode with ECI* を正しく使用すれば、世界中のスキャナーで読み取り可能な状態を保てます。

このチュートリアルでは、Aspose.Barcode Python ライブラリを使って **Code128 バーコード** を **生成** する完全な実行可能サンプルを順を追って解説します。拡張 codetext に ECI セグメントを組み込む方法、各行の意味、そして一般的な落とし穴を回避するプロのコツまで網羅しています。最後まで読めば、すぐに実行できるスクリプトと概念の理解、そして実務で役立つヒントが手に入ります。

## 学べること

- **Aspose.Barcode Python** の環境設定方法  
- プレーンな数字と UTF‑8 テキストを ECI セグメントで混在させた拡張 codetext の作成方法  
- `BarCodeGenerator` を設定して **Code128 バーコード** を出力する手順  
- 画像の保存と結果の検証方法  
- 文字セットやシンボロジー別のエッジケース処理

外部ドキュメントは不要です。必要な情報はすべてここにあります。

## 前提条件

作業を始める前に、以下を確認してください。

1. **Python 3.8 以上** がインストール済み（ライブラリは 3.6 以降をサポート）  
2. 有効な **Aspose.Barcode for Python via .NET** ライセンス、または評価版の透かしが許容できること  
3. パッケージをインストールできるターミナルまたは IDE（`pip` が使える環境）  
4. 生成した PNG を保存するフォルダー（コード中の `YOUR_DIRECTORY` を置き換える）

これらが見慣れないものであっても心配はいりません。パッケージのインストールは 1 行で完了し、残りは純粋な Python です。

---

## 手順 1: Aspose.Barcode for Python をインストール

ターミナルで次のコマンドを実行してください。

```bash
pip install aspose-barcode
```

> **プロ tip:** 仮想環境（`python -m venv venv`）を使うと依存関係がプロジェクトごとに分離でき、他のプロジェクトとのバージョン衝突を防げます。

パッケージは内部で .NET ランタイムを呼び出すため、最初のインポート時に少し時間がかかることがあります。

---

## 手順 2: 拡張 Codetext Builder を作成

**barcode with ECI** のコア機能は `ExtCodetextBuilder` にあります。プレーンなセグメントと ECI エンコードされた部分を連結でき、スキャナーが Unicode テキストを正しく解釈できるようになります。

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**なぜ clear が必要か？**  
ループ内でビルダーを再利用すると、前回のデータが残って次のバーコードが壊れる可能性があります。`clear()` を呼び出すことで毎回新しい状態から始められます。

---

## 手順 3: プレーンな数値セグメントを追加

ほとんどの Code128 バーコードは数値識別子（注文番号や製品 ID など）で始まります。*プレーン* セグメントとして追加すると、エンコードがシンプルかつコンパクトになります。

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

このメソッドは数字に最適な Code128 のサブセット（A、B、C）を自動で選択し、可能な限り小さなバーコードを生成します。

---

## 手順 4: ロシア語テキストを含む ECI セグメントを挿入

ここがポイントです：ロシア語文字列（`Пример текста`）を埋め込むには ECI マーカーが必須です。マーカーが無いと多くのスキャナーはバイト列を誤って解釈し、文字化けします。

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**何が起きているか？**  
- `26` は「以降のバイトは UTF‑8 エンコードである」ことを示す ECI 値です。  
- ライブラリが変換を内部で処理するので、Unicode 文字列をそのまま渡すだけで OK。  
- スキャナーは ECI を検知すると UTF‑8 に切り替え、ロシア語テキストを正しく表示します。

> **よくある質問:** *別の言語が必要な場合は？*  
> 適切な ECI 値（例: ISO‑8859‑1 は 27、ISO‑8859‑2 は 28）を使用します。対応表は Aspose のドキュメントにありますが、使い方は同じです。

---

## 手順 5: 完全な拡張 Codetext を取得

両方のセグメントを追加したら、結合された文字列を取得します。これが `BarCodeGenerator` の `code_text` に設定する値です。

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

コンソールに出力される例は次のようになります。

```
Extended codetext: 1234567890[ECI26]Пример текста
```

`[ECI26]` トークンは最終画像では見えませんが、正しいデコードに不可欠です。

---

## 手順 6: Code128 用に BarCodeGenerator を設定

ここで全体を結びつけます。`BarCodeGenerator` が画像生成の主役です。

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**`x_dimension` を設定する理由**  
モジュール幅を小さくするとバーコードがコンパクトになり、限られたスペースに有利です。プリンターの DPI に合わせて調整してください。

---

## 手順 7: バーコード画像を保存

最後に PNG をディスクに書き出します。拡張子を変えれば JPEG、BMP、SVG でも出力可能です。

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

任意のビューアで画像を開くと、最新のリーダーでスキャンした際に `1234567890Пример текста` が返ってくるきれいな Code128 バーコードが確認できます。

---

## 完全スクリプト – コピーして実行

以下が単体で動作する完全スクリプトです。`generate_code128.py` という名前で保存し、`YOUR_DIRECTORY` を適切なパスに置き換えてから `python generate_code128.py` を実行してください。

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**期待されるコンソール出力**（例）:

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

`extended_code.png` を開くと、実運用に耐えうる鮮明な Code128 バーコードが表示されます。

## 深掘り: ECI メカニズムの仕組み

ECI（Extended Channel Interpretation）は QR コードやバーコード規格に組み込まれた、データストリーム途中で文字エンコーディングを変更するための制御情報です。Code128 における流れは次の通りです。

1. バーコードはデフォルトで ISO‑8859‑1 モードで開始  
2. スキャナーが ECI 制御コード（本例では値 26）に遭遇すると UTF‑8 に切り替え  
3. 以降のバイトは UTF‑8 として解釈され、別の ECI が現れるかデータ終端になるまで続く

`add_eci_codetext(26, ...)` を呼び出すことで、Aspose が内部的に正しい制御文字を挿入します。ECI を省略すると、ロシア語テキストはデフォルトコードページでエンコードされ、ほとんどのスキャナーで読めない文字化けが発生します。

---

## エッジケースの取り扱い

### 1. 複数の ECI セグメント

日本語とアラビア語の両方を埋め込みたい場合は、異なる ECI 値で `add_eci_codetext` を複数回呼び出します。ビルダーは追加順に連結します。

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. 長さの制限

Code128 の最大文字数は 2 023 文字ですが、スキャナーのメモリ制限により実用的にはもっと少なくなることが多いです。制限を超えると `ArgumentException` がスローされます。ECI セグメントを混在させる場合は、総文字数を 1 000 未満に抑えると安全です。

### 3. 古いスキャナーとの互換性

一部のレガシースキャナーは ECI 制御コードを無視します。そのような環境では、数値部と Unicode 部を別々のバーコードに分割するか、ECI 対応が広い QR コードなど別シンボロジーを選択してください。

---

## 本番環境でのプロ tip

- **早めにライセンスを適用**: 評価版は小さな透かしが入ります。`BarCodeGenerator` を最初に呼び出す前にライセンスファイルをロードしましょう。  
- **バッチ処理**: 数千本のバーコードを生成する場合は、`ExtCodetextBuilder` のインスタンスを使い回し、各ループで `clear()` するだけで済みます。  
- **スレッド安全性**: 各スレッドは独自の `BarCodeGenerator` インスタンスを保持してください。オブジェクトはスレッドセーフではありません。  
- **パフォーマンス**: `x_dimension` を極端に小さくすると描画時間が増加します。高スループットが必要な場合はハードウェアでベンチマークしてください。  
- **テスト**: 生成したバーコードは必ずモバイルアプリとハードウェアスキャナーの 2 種類以上で走査し、ECI 処理が正しく行われているか確認しましょう。
