## 多言語 PDF417 バーコード Java とは？
PDF417 is a stacked linear barcode format that can encode large amounts of data, including Unicode characters. When a PDF417 barcode contains characters from languages such as Chinese or Turkish, you need a decoder that understands Unicode and can map those glyphs correctly. The Aspose.BarCode Java API provides built‑in multilingual support, allowing you to read and interpret these characters without additional configuration.

## よくある問題と解決策
- **文字のレンダリングが正しくない** – ソース画像が高解像度であること、`BarcodeReader` が `BarcodeReadType.PDF417` に設定されていることを確認してください。  
- **ライセンスが見つからない** – `Aspose.BarCode.lic` ファイルがクラスパスに配置されているか、`License.setLicense()` でプログラム的に設定されていることを確認してください。  
- **大量バッチでのパフォーマンス** – 完全な精度が必須でない場合、`BarcodeReader.setReadQuality(BarcodeReadQuality.Low)` を使用して処理速度を向上させます。  

## よくある質問

**Q: 単一シンボル内で複数言語が混在した PDF417 バーコードをデコードできますか？**  
A: はい。Aspose.BarCode の Unicode エンジンは混在言語データを自動的に処理します。

**Q: デコードできる文字数に制限はありますか？**  
A: PDF417 は最大 1,850 文字のテキストを格納できます。バーコードが損傷していない限り、ライブラリは完全なデコード文字列を返します。

**Q: トルコ語文字のために特別なオプションを設定する必要がありますか？**  
A: 特別なオプションは不要です。デフォルトの Unicode 処理でトルコ語文字はそのままサポートされます。

**Q: PDF417 バーコードを含むパスワード保護された PDF ファイルをどう扱いますか？**  
A: `BarCodeReader` の `PdfDocument` オブジェクトを受け取るオーバーロードを使用し、適切なパスワードで PDF をロードします。

**Q: このライブラリは Android と互換性がありますか？**  
A: はい、Aspose.BarCode for Java は Android Studio プロジェクトで、最低 API レベル 21 で動作します。
