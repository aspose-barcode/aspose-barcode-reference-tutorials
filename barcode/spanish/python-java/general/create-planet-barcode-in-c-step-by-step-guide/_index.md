# Crear código de barras Planet en C# – Guía paso a paso

¿Alguna vez necesitaste **crear código de barras planet** pero no estabas seguro por dónde empezar? En este tutorial verás exactamente **cómo generar código de barras planet** usando la biblioteca Aspose.Barcode, con opciones de altura predeterminada y personalizada.

Recorreremos todo el proceso—desde instalar el paquete NuGet hasta ajustar la altura de la barra y guardar archivos PNG. Al final tendrás cuatro imágenes listas para usar: dos códigos de barras Planet (auto y altura de 100 píxeles) y dos códigos de barras RM4SCC para comparación. Sin rodeos, solo código práctico que puedes insertar en cualquier proyecto .NET.

## Requisitos previos

Antes de comenzar, asegúrate de tener:

* .NET 6.0 o posterior (el código también funciona en .NET Framework 4.7+)
* Visual Studio 2022 (o cualquier IDE que prefieras)
* El paquete NuGet **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Una carpeta en disco donde se puedan escribir los archivos PNG generados (reemplaza `YOUR_DIRECTORY` en el código).

Si alguno de estos conceptos te resulta desconocido, no te preocupes—instalar un paquete NuGet es tan fácil como hacer clic derecho en el proyecto → *Manage NuGet Packages* → buscar “Aspose.Barcode” → *Install*.

## Paso 1 – Crear un proyecto de consola mínimo

Primero, vamos a crear una pequeña aplicación de consola para ejecutar el código del código de barras de forma aislada.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Consejo profesional:** Mantén el método `Main` ordenado; delega el trabajo pesado a métodos separados. Así el código es más fácil de probar y de leer más adelante.

## Paso 2 – **Crear código de barras Planet** con altura predeterminada

Ahora realmente **creamos código de barras planet** usando la altura predeterminada (calculada automáticamente). `XDimension` controla el ancho de un módulo único, mientras que la altura de la barra la deja la biblioteca.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*¿Por qué funciona esto?*  
`EncodeTypes.Planet` indica a Aspose que queremos la simbología Planet, y `"123456"` es la carga de datos. Al no establecer `BarHeight`, el motor elige una altura que cumple con la especificación automáticamente.

### Salida esperada

Abre `PlanetBarHeightAuto.png` y deberías ver un código de barras Planet limpio, aproximadamente 30 mm de alto (el tamaño exacto depende del DPI del escritor PNG). No se requiere ajuste manual de altura.

## Paso 3 – **Cómo generar código de barras Planet** con una altura personalizada de 100 píxeles

Si tu diseño requiere una altura fija—por ejemplo, 100 píxeles—simplemente establece la propiedad `BarHeight` antes de guardar.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Observa la línea adicional `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Eso fuerza al código de barras a estirarse (o encogerse) a exactamente 100 píxeles de alto, sin importar la longitud de los datos.

> **Cuidado:** Establecer una altura demasiado pequeña puede hacer que las barras se fusionen, dejando el código de barras ilegible. Una regla práctica segura es mantener la altura al menos tres veces la X‑dimension.

## Paso 4 – Generar códigos de barras RM4SCC para comparación

A veces necesitas comparar Planet con otras simbologías postales. RM4SCC es otro formato común; el código a continuación replica los pasos que usamos para Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Ejecutar el programa completo ahora produce **cuatro** archivos PNG:

| Archivo | Simbología | Altura |
|---------|------------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Puedes abrir cada imagen en cualquier visor para confirmar las dimensiones y la calidad visual.

## Paso 5 – Verificar los códigos de barras programáticamente (Opcional)

Si deseas estar completamente seguro de que los códigos de barras generados son escaneables, Aspose también ofrece un componente **reader**. Aquí tienes una rápida comprobación de que los datos codificados coinciden con nuestra cadena original.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Ejecutar `VerifyBarcodes()` muestra `Decoded value: 123456`, confirmando que la altura personalizada no rompió la codificación.

## Problemas comunes y consejos

| Problema | Por qué ocurre | Solución |
|----------|----------------|----------|
| El código de barras aparece demasiado delgado | X‑dimension establecida demasiado baja (p. ej., 1 píxel) | Incrementa `XDimension.Pixels` a 3‑5 para la mayoría de impresoras |
| La imagen está borrosa | Guardado como PNG con DPI predeterminado (96) en pantallas de alta resolución | Usa `ImageOptions` para establecer `ResolutionX` y `ResolutionY` a 300 DPI |
| La altura no se aplica | `BarHeight` configurado *después* de llamar a `Save()` | Siempre establece `BarHeight` **antes** de `Save()` |
| Ruta de archivo no válida | `YOUR_DIRECTORY` no existe | Crea la carpeta programáticamente: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
