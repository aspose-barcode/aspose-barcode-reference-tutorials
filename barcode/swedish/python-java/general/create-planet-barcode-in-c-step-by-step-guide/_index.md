# Skapa Planet-streckkod i C# – Steg‑för‑steg guide

Har du någonsin behövt **skapa planet-streckkod** men var osäker på var du ska börja? I den här handledningen kommer du att se exakt **hur man genererar planet-streckkod** med hjälp av Aspose.Barcode-biblioteket, med både standard‑ och anpassade höjdalternativ.

Vi går igenom hela processen—från att installera NuGet‑paketet till att justera stapelhöjden och spara PNG‑filer. I slutet har du fyra färdiga bilder: två Planet‑streckkoder (automatisk och 100‑pixel hög) och två RM4SCC‑streckkoder för jämförelse. Inga onödiga detaljer, bara praktisk kod som du kan klistra in i vilket .NET‑projekt som helst.

## Förutsättningar

* .NET 6.0 eller senare (koden fungerar även på .NET Framework 4.7+).
* Visual Studio 2022 (eller någon IDE du föredrar).
* Det **Aspose.Barcode for .NET** NuGet‑paketet  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* En mapp på disken där de genererade PNG‑filerna kan skrivas (ersätt `YOUR_DIRECTORY` i koden).

Om någon av dessa låter obekant, oroa dig inte—att installera ett NuGet‑paket är lika enkelt som att högerklicka på projektet → *Manage NuGet Packages* → sök efter “Aspose.Barcode” → *Install*.

## Steg 1 – Skapa ett minimalt konsolprojekt

Först, låt oss skapa en liten konsolapp så att vi kan köra streckkodskoden isolerat.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Proffstips:** Håll `Main`‑metoden ren; delegera det tunga arbetet till separata metoder. Det gör koden enklare att testa och läsa senare.

## Steg 2 – **Skapa Planet-streckkod** med standardhöjd

Nu **skapar vi planet-streckkod** med den standard (automatiskt beräknade) höjden. `XDimension` styr bredden på en enskild modul, medan stapelhöjden lämnas åt biblioteket.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Varför fungerar detta?*  
`EncodeTypes.Planet` talar om för Aspose att vi vill ha Planet‑symbologi, och `"123456"` är datapayloaden. Genom att inte sätta `BarHeight` väljer motorn automatiskt en höjd som uppfyller specifikationen.

### Förväntat resultat

Öppna `PlanetBarHeightAuto.png` så bör du se en ren Planet‑streckkod, ungefär 30 mm hög (den exakta storleken beror på DPI‑inställningarna för PNG‑skrivaren). Ingen manuell justering av höjden behövs.

## Steg 3 – **Hur man genererar Planet-streckkod** med en anpassad 100‑pixelhöjd

Om din design kräver en fast höjd—t.ex. 100 pixel—sätter du helt enkelt `BarHeight`‑egenskapen innan du sparar.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Observera den extra raden `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Den tvingar streckkoden att sträcka sig (eller krympa) till exakt 100 pixel hög, oavsett datalängd.

> **Observera:** Att sätta en höjd som är för liten kan få staplarna att smälta samman, vilket gör streckkoden oläslig. En säker tumregel är att hålla höjden minst tre gånger X‑dimensionen.

## Steg 4 – Generera RM4SCC‑streckkoder för jämförelse

Ibland behöver du jämföra Planet med andra post‑symbologier. RM4SCC är ett annat vanligt format; koden nedan speglar stegen vi använde för Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Att köra hela programmet nu genererar **fyra** PNG‑filer:

| Fil | Symbologi | Höjd |
|------|-----------|------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Du kan öppna varje bild i någon visare för att bekräfta dimensionerna och den visuella kvaliteten.

## Steg 5 – Verifiera streckkoderna programatiskt (valfritt)

Om du vill vara extra säker på att de genererade streckkoderna kan läsas, erbjuder Aspose även en **reader**‑komponent. Här är en snabb kontroll att den kodade datan matchar vår ursprungliga sträng.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Att köra `VerifyBarcodes()` skriver ut `Decoded value: 123456`, vilket bekräftar att den anpassade höjden inte förstörde kodningen.

## Vanliga fallgropar & tips

| Problem | Varför det händer | Lösning |
|---------|-------------------|--------|
| Streckkoden verkar för tunn | X‑dimension är satt för låg (t.ex. 1 pixel) | Öka `XDimension.Pixels` till 3‑5 för de flesta skrivare |
| Bilden är suddig | Sparas som PNG med standard‑DPI (96) på högupplösta skärmar | Använd `ImageOptions` för att sätta `ResolutionX` och `ResolutionY` till 300 DPI |
| Höjden tillämpas inte | `BarHeight` satt *efter* anropet `Save()` | Sätt alltid `BarHeight` **före** `Save()` |
| Filvägen ogiltig | `YOUR_DIRECTORY` finns inte | Skapa mappen programatiskt: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
