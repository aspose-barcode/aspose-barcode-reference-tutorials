# Créer un code‑barres Planet en C# – Guide étape par étape

Vous avez déjà eu besoin de **créer un code‑barres planet** mais vous ne saviez pas par où commencer ? Dans ce tutoriel, vous verrez exactement **comment générer un code‑barres planet** en utilisant la bibliothèque Aspose.Barcode, avec des options de hauteur par défaut et personnalisée.

Nous parcourrons l’ensemble du processus — de l’installation du package NuGet à l’ajustement de la hauteur des barres et à l’enregistrement des fichiers PNG. À la fin, vous disposerez de quatre images prêtes à l’emploi : deux codes‑barres Planet (hauteur automatique et 100 pixels) et deux codes‑barres RM4SCC pour la comparaison. Pas de fioritures, juste du code pratique que vous pouvez intégrer dans n’importe quel projet .NET.

## Prérequis

* .NET 6.0 ou ultérieur (le code fonctionne également sur .NET Framework 4.7+)
* Visual Studio 2022 (ou tout IDE de votre choix)
* Le package NuGet **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Un dossier sur le disque où les fichiers PNG générés peuvent être écrits (remplacez `YOUR_DIRECTORY` dans le code).

Si l’un de ces éléments vous est inconnu, ne vous inquiétez pas — installer un package NuGet est aussi simple que de faire un clic droit sur le projet → *Manage NuGet Packages* → rechercher “Aspose.Barcode” → *Install*.

## Étape 1 – Créer un projet console minimal

Tout d’abord, créons une petite application console afin de pouvoir exécuter le code du code‑barres isolément.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Astuce :** Gardez la méthode `Main` propre ; déléguez le travail lourd à des méthodes séparées. Cela rend le code plus facile à tester et à lire plus tard.

## Étape 2 – **Créer un code‑barres Planet** avec hauteur par défaut

Nous allons maintenant réellement **créer un code‑barres planet** en utilisant la hauteur par défaut (calculée automatiquement). Le `XDimension` contrôle la largeur d’un module unique, tandis que la hauteur des barres est laissée à la bibliothèque.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Pourquoi cela fonctionne‑t‑il ?*  
`EncodeTypes.Planet` indique à Aspose que nous voulons la symbologie Planet, et `"123456"` est la charge de données. En ne définissant pas `BarHeight`, le moteur choisit automatiquement une hauteur qui satisfait la spécification.

### Résultat attendu

Ouvrez `PlanetBarHeightAuto.png` et vous devriez voir un code‑barres Planet net, d’environ 30 mm de hauteur (la taille exacte dépend du DPI du générateur PNG). Aucun ajustement manuel de la hauteur n’est nécessaire.

## Étape 3 – **Comment générer un code‑barres Planet** avec une hauteur personnalisée de 100 pixels

Si votre conception nécessite une hauteur fixe — par exemple, 100 pixels — il suffit de définir la propriété `BarHeight` avant d’enregistrer.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Remarquez la ligne supplémentaire `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Elle force le code‑barres à s’étirer (ou se rétrécir) à exactement 100 pixels de hauteur, quel que soit la longueur des données.

> **Attention :** Définir une hauteur trop petite peut entraîner la fusion des barres, rendant le code‑barres illisible. Une règle de prudence consiste à garder la hauteur d’au moins trois fois la X‑dimension.

## Étape 4 – Générer des codes‑barres RM4SCC pour la comparaison

Parfois, vous devez comparer Planet avec d’autres symbologies postales. RM4SCC est un autre format courant ; le code ci‑dessous reproduit les étapes que nous avons utilisées pour Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

L’exécution du programme complet produit maintenant **quatre** fichiers PNG :

| Fichier | Symbologie | Hauteur |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Vous pouvez ouvrir chaque image dans n’importe quel visualiseur pour confirmer les dimensions et la qualité visuelle.

## Étape 5 – Vérifier les codes‑barres programmatiquement (facultatif)

Si vous voulez être absolument certain que les codes‑barres générés sont lisibles, Aspose propose également un composant **reader**. Voici une vérification rapide pour s’assurer que les données encodées correspondent à notre chaîne d’origine.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

L’exécution de `VerifyBarcodes()` affiche `Decoded value: 123456`, confirmant que la hauteur personnalisée n’a pas cassé l’encodage.

## Pièges courants & astuces

| Problème | Pourquoi cela se produit | Solution |
|-------|----------------|-----|
| Le code‑barres apparaît trop fin | X‑dimension réglée trop basse (ex. 1 pixel) | Augmentez `XDimension.Pixels` à 3‑5 pour la plupart des imprimantes |
| L’image est floue | Enregistrement en PNG avec DPI par défaut (96) sur des écrans haute résolution | Utilisez `ImageOptions` pour définir `ResolutionX` et `ResolutionY` à 300 DPI |
| Hauteur non appliquée | `BarHeight` défini *après* l’appel à `Save()` | Toujours définir `BarHeight` **avant** `Save()` |
| Chemin de fichier invalide | `YOUR_DIRECTORY` n’existe pas | Créez le dossier programmatique : `Directory.CreateDirectory("YOUR_DIRECTORY");` |
