# Generate Code128 Barcode with Aspose.Barcode Python – Full Guide

Ever needed to **generate Code128 barcode** in a Python project but weren’t sure how to embed non‑ASCII characters? You’re not alone. Many developers hit a wall when they try to mix numeric data with Unicode text—especially Russian or Chinese—inside a single barcode. The good news? Aspose.Barcode for Python makes it painless, and with the right use of *barcode with ECI* you can keep everything legible for scanners worldwide.

In this tutorial we’ll walk through a complete, runnable example that shows exactly how to **generate Code128 barcode** using the Aspose.Barcode Python library, how to craft an extended codetext that includes an ECI segment, and why each line matters. By the end you’ll have a ready‑to‑run script, a clear understanding of the underlying concepts, and a few pro tips to avoid common pitfalls.

## What You’ll Learn

- Setting up **Aspose.Barcode Python** in your environment.  
- Building an extended codetext string that mixes plain numbers with UTF‑8 text via an ECI segment.  
- Configuring the `BarCodeGenerator` to output a **Code128 barcode**.  
- Saving the image and verifying the result.  
- Edge‑case handling for different character sets and barcode symbologies.

No external documentation is required—everything you need is right here.

## Step 2: Create an Extended Codetext Builder

The heart of **barcode with ECI** support lies in the `ExtCodetextBuilder`. It lets you concatenate plain segments with ECI‑encoded parts, ensuring scanners interpret the Unicode text correctly.

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**Why clear?**  
If you reuse the builder in a loop, lingering data could corrupt the next barcode. Calling `clear()` guarantees a fresh slate each time.

---

## Step 3: Add a Plain Numeric Segment

Most Code128 barcodes start with a numeric identifier—think of an order number or product ID. Adding it as a *plain* segment keeps the encoding simple and compact.

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

The method automatically picks the optimal Code128 subset (A, B, or C) for the digits, resulting in the smallest possible barcode.

---

## Step 4: Insert an ECI Segment with Russian Text

Now for the tricky part: embedding Russian characters (`Пример текста`). Without an ECI marker, most scanners would interpret those bytes incorrectly, producing garbled output.

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**What’s happening?**  
- `26` tells the barcode that the following bytes use UTF‑8 encoding.  
- The library handles the conversion, so you simply pass a Unicode string.  
- When the scanner reads the barcode, it sees the ECI, switches to UTF‑8, and renders the Russian text correctly.

> **Common question:** *What if I need a different language?*  
> Use the appropriate ECI value (e.g., 27 for ISO‑8859‑1, 28 for ISO‑8859‑2). A quick lookup table is available in the Aspose docs, but the pattern stays the same.

---

## Step 5: Retrieve the Full Extended Codetext

Once both segments are added, we pull the combined string. This is the value you’ll assign to the generator’s `code_text`.

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

The printed output looks something like:

```
Extended codetext: 1234567890[ECI26]Пример текста
```

The `[ECI26]` token is invisible in the final image but crucial for proper decoding.

---

## Step 6: Configure the BarCodeGenerator for Code128

Now we tie everything together. The `BarCodeGenerator` is the workhorse that renders the image.

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**Why set `x_dimension`?**  
A smaller module width yields a more compact barcode—useful for limited space. Adjust it based on your printer’s DPI.

## Step 7: Save the Barcode Image

Finally, write the PNG to disk. You can also export to JPEG, BMP, or even SVG by changing the file extension.

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

Open the image with any viewer; you should see a clean Code128 barcode that, when scanned with a modern reader, returns `1234567890Пример текста`.

## Full Script – Paste and Run

Below is the complete, self‑contained script. Copy it into a file named `generate_code128.py`, adjust `YOUR_DIRECTORY`, and execute `python generate_code128.py`.

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**Expected output** (console):

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

Open `extended_code.png` and you’ll see a crisp Code128 barcode ready for production use.

## Deep Dive: How the ECI Mechanism Works

ECI (Extended Channel Interpretation) is a part of the QR and barcode standards that signals a change in character encoding mid‑stream. In the context of Code128:

1. The barcode starts in its default ISO‑8859‑1 mode.  
2. When the scanner encounters the ECI control code (value 26 in our example), it switches to UTF‑8.  
3. All subsequent bytes are interpreted as UTF‑8 until another ECI or the end of the data.

Because we explicitly set `add_eci_codetext(26, ...)`, Aspose inserts the correct control characters behind the scenes. If you omit the ECI, the Russian text would be encoded using the default code page, typically resulting in unreadable garble on most scanners.

## Handling Edge Cases

### 1. Multiple ECI Segments

If you need to embed both Japanese and Arabic text, you can call `add_eci_codetext` multiple times with different ECI values. The builder concatenates them in the order you add them.

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. Length Limits

Code128 can encode up to 2 023 characters, but the practical limit is often lower due to scanner memory. If you exceed it, the generator throws `ArgumentException`. To stay safe, keep the total length under 1 000 characters when mixing ECI segments.

### 3. Compatibility with Older Scanners

Some legacy scanners ignore ECI control codes. In such environments, you might need to generate separate barcodes—one for the numeric part, another for the Unicode text—or use a different symbology like QR that has broader ECI support.

## Pro Tips for Production Use

- **License early:** The evaluation version adds a small watermark. Apply your license file before the first `BarCodeGenerator` call to avoid it.
- **Batch processing:** When generating thousands of barcodes, reuse a single `ExtCodetextBuilder` instance—just call `clear()` each loop iteration.
- **Thread safety:** Each thread should own its own `BarCodeGenerator` instance; the objects are not thread‑safe.
- **Performance:** Setting `x_dimension` too low can increase rendering time. Benchmark with your target hardware if you’re in a high‑throughput scenario.
- **Testing:** Always scan the generated barcode with at least two different readers (mobile app and hardware scanner) to verify ECI handling.
