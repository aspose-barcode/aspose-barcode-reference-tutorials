# Aspose.Barcode Python으로 Code128 바코드 생성 – 전체 가이드

Python 프로젝트에서 **Code128 바코드**를 생성해야 했지만 비‑ASCII 문자를 어떻게 삽입해야 할지 몰라 고민한 적이 있나요? 당신만 그런 것이 아닙니다. 많은 개발자들이 숫자 데이터와 유니코드 텍스트(특히 러시아어나 중국어)를 하나의 바코드에 섞으려 할 때 벽에 부딪힙니다. 좋은 소식은? Aspose.Barcode for Python을 사용하면 이 과정이 매우 간단해지고, *barcode with ECI*를 올바르게 활용하면 전 세계 스캐너가 내용을 올바르게 읽을 수 있습니다.

이 튜토리얼에서는 Aspose.Barcode Python 라이브러리를 사용해 **Code128 바코드**를 **생성**하는 완전한 실행 예제를 단계별로 살펴보고, ECI 구간을 포함한 확장 코드텍스트를 만드는 방법과 각 라인의 의미를 설명합니다. 튜토리얼을 마치면 바로 실행 가능한 스크립트를 얻고, 핵심 개념을 명확히 이해하며, 흔히 발생하는 실수를 피할 수 있는 몇 가지 팁도 습득하게 됩니다.

## 배울 내용

- 환경에 **Aspose.Barcode Python**을 설정하는 방법  
- 평문 숫자와 UTF‑8 텍스트를 ECI 구간으로 섞어 만든 확장 코드텍스트 문자열 만들기  
- `BarCodeGenerator`를 구성해 **Code128 바코드**를 출력하는 방법  
- 이미지 저장 및 결과 확인 방법  
- 다양한 문자 집합 및 바코드 심볼에 대한 엣지 케이스 처리

외부 문서는 필요 없습니다—여기에 모든 것이 준비되어 있습니다.

## 사전 요구 사항

1. **Python 3.8+**이 설치되어 있어야 합니다(라이브러리는 3.6 이상을 지원).  
2. 유효한 **Aspose.Barcode for Python via .NET** 라이선스가 있거나 평가용 워터마크 사용에 동의해야 합니다.  
3. 패키지를 설치할 수 있는 터미널 또는 IDE(`pip` 사용 가능).  
4. 생성된 PNG가 저장될 폴더(`YOUR_DIRECTORY`를 코드에서 교체)  

위 항목 중 익숙하지 않은 것이 있더라도 걱정하지 마세요—패키지 설치는 한 줄 명령으로 끝나고 나머지는 순수 Python만으로 진행됩니다.

## Step 1: Install Aspose.Barcode for Python

터미널을 열고 다음을 실행하세요:

```bash
pip install aspose-barcode
```

> **Pro tip:** `python -m venv venv`와 같은 가상 환경을 사용하면 종속성을 격리할 수 있습니다. 이렇게 하면 다른 프로젝트와의 버전 충돌도 방지됩니다.

패키지는 내부적으로 .NET 런타임을 포함하므로 첫 번째 import 시 네이티브 라이브러리를 로드하는 데 약간의 시간이 걸릴 수 있습니다.

---

## Step 2: Create an Extended Codetext Builder

**barcode with ECI** 지원의 핵심은 `ExtCodetextBuilder`에 있습니다. 이 빌더를 사용하면 평문 구간과 ECI‑인코딩 구간을 자유롭게 연결할 수 있어 스캐너가 유니코드 텍스트를 올바르게 해석하도록 보장합니다.

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**왜 clear()?**  
루프 안에서 빌더를 재사용하면 이전 데이터가 남아 다음 바코드가 손상될 수 있습니다. `clear()`를 호출하면 매번 새 상태를 보장합니다.

---

## Step 3: Add a Plain Numeric Segment

대부분의 Code128 바코드는 숫자 식별자로 시작합니다—예를 들어 주문 번호나 제품 ID가 될 수 있습니다. 이를 *plain* 구간으로 추가하면 인코딩이 간단하고 압축됩니다.

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

이 메서드는 숫자에 가장 적합한 Code128 서브셋(A, B, C)을 자동으로 선택해 가능한 가장 작은 바코드를 생성합니다.

---

## Step 4: Insert an ECI Segment with Russian Text

이제 어려운 부분입니다: 러시아어 문자(`Пример текста`)를 삽입합니다. ECI 마커가 없으면 대부분의 스캐너가 바이트를 잘못 해석해 깨진 문자열이 출력됩니다.

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**무슨 일이 일어나나요?**  
- `26`은 이후 바이트가 UTF‑8 인코딩임을 바코드에 알립니다.  
- 라이브러리가 변환을 자동으로 처리하므로 유니코드 문자열만 전달하면 됩니다.  
- 스캐너가 바코드를 읽을 때 ECI를 감지하고 UTF‑8로 전환해 러시아어 텍스트를 올바르게 표시합니다.

> **Common question:** *다른 언어가 필요하면 어떻게 하나요?*  
> 해당 언어에 맞는 ECI 값을 사용하면 됩니다(예: ISO‑8859‑1은 27, ISO‑8859‑2는 28). 자세한 값은 Aspose 문서에 표가 제공되며, 사용 방법은 동일합니다.

---

## Step 5: Retrieve the Full Extended Codetext

두 구간을 모두 추가한 뒤 결합된 문자열을 가져옵니다. 이 문자열을 `BarCodeGenerator`의 `code_text`에 할당하면 됩니다.

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

출력 예시는 다음과 같습니다:

```
Extended codetext: 1234567890[ECI26]Пример текста
```

`[ECI26]` 토큰은 최종 이미지에서는 보이지 않지만 올바른 디코딩을 위해 필수적입니다.

---

## Step 6: Configure the BarCodeGenerator for Code128

이제 모든 요소를 연결합니다. `BarCodeGenerator`는 실제 이미지를 렌더링하는 핵심 객체입니다.

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**왜 `x_dimension`을 설정하나요?**  
모듈 폭을 작게 하면 바코드가 더 컴팩트해집니다—공간이 제한된 경우 유용합니다. 프린터 DPI에 맞춰 적절히 조정하세요.

## Step 7: Save the Barcode Image

PNG 파일을 디스크에 저장합니다. 파일 확장자를 JPEG, BMP, SVG 등으로 바꾸면 다른 포맷으로도 내보낼 수 있습니다.

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

이미지를 아무 뷰어로 열어보면 깔끔한 Code128 바코드가 보이며, 최신 리더기로 스캔하면 `1234567890Пример текста`가 반환됩니다.

## Full Script – Paste and Run

아래는 완전하고 독립적인 스크립트 전체입니다. `generate_code128.py`라는 파일에 복사하고, `YOUR_DIRECTORY`를 실제 경로로 바꾼 뒤 `python generate_code128.py`를 실행하세요.

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**예상 출력** (콘솔):

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

`extended_code.png`를 열면 생산 환경에서 바로 사용할 수 있는 선명한 Code128 바코드를 확인할 수 있습니다.

## Deep Dive: How the ECI Mechanism Works

ECI(Extended Channel Interpretation)는 QR 및 바코드 표준의 일부로, 스트림 중간에 문자 인코딩이 변경됨을 알리는 역할을 합니다. Code128에서의 동작은 다음과 같습니다.

1. 바코드는 기본 ISO‑8859‑1 모드로 시작합니다.  
2. 스캐너가 ECI 제어 코드(예시에서는 값 26)를 만나면 UTF‑8로 전환합니다.  
3. 이후 바이트는 모두 UTF‑8로 해석되며, 다른 ECI가 나오거나 데이터가 끝날 때까지 지속됩니다.

`add_eci_codetext(26, ...)`를 명시적으로 호출했기 때문에 Aspose가 내부적으로 올바른 제어 문자를 삽입합니다. ECI를 생략하면 러시아어 텍스트가 기본 코드 페이지로 인코딩돼 대부분의 스캐너에서 읽을 수 없는 잡음이 됩니다.

## Handling Edge Cases

### 1. Multiple ECI Segments

일본어와 아랍어 같이 서로 다른 언어를 동시에 삽입해야 할 경우, 서로 다른 ECI 값을 사용해 `add_eci_codetext`를 여러 번 호출하면 됩니다. 빌더는 추가된 순서대로 구간을 연결합니다.

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. Length Limits

Code128은 최대 2 023자를 인코딩할 수 있지만, 스캐너 메모리 제한으로 실제 한계는 더 낮을 수 있습니다. 제한을 초과하면 `ArgumentException`이 발생합니다. ECI 구간을 섞을 때는 전체 길이를 1 000자 이하로 유지하는 것이 안전합니다.

### 3. Compatibility with Older Scanners

일부 구형 스캐너는 ECI 제어 코드를 무시합니다. 이런 경우 숫자 부분과 유니코드 텍스트를 각각 별도의 바코드로 생성하거나, ECI 지원이 더 넓은 QR 같은 다른 심볼을 사용하는 것이 좋습니다.

## Pro Tips for Production Use

- **License early:** 평가 버전은 작은 워터마크를 삽입합니다. 첫 `BarCodeGenerator` 호출 전에 라이선스 파일을 적용해 워터마크를 없애세요.  
- **Batch processing:** 수천 개의 바코드를 생성할 때는 `ExtCodetextBuilder` 인스턴스를 하나만 재사용하고, 각 루프마다 `clear()`를 호출하면 효율적입니다.  
- **Thread safety:** 각 스레드는 자체 `BarCodeGenerator` 인스턴스를 가져야 하며, 객체는 스레드‑안전하지 않습니다.  
- **Performance:** `x_dimension`을 너무 작게 설정하면 렌더링 시간이 늘어날 수 있습니다. 고처리량 환경에서는 목표 하드웨어에서 벤치마크해 최적값을 찾으세요.  
- **Testing:** 생성된 바코드를 최소 두 종류의 리더(모바일 앱 및 하드웨어 스캐너)로 반드시 스캔해 ECI 처리가 정상인지 확인하세요.
