# Создание штрих‑кода Planet в C# – пошаговое руководство

Когда‑нибудь нужно было **создать штрих‑код Planet**, но не знали, с чего начать? В этом руководстве вы увидите, **как генерировать штрих‑код Planet** с помощью библиотеки Aspose.Barcode, используя как параметры высоты по умолчанию, так и пользовательские.

Мы пройдем весь процесс — от установки пакета NuGet до настройки высоты полос и сохранения PNG‑файлов. В конце у вас будет четыре готовых изображения: два штрих‑кода Planet (авто и высотой 100 пикселей) и два штрих‑кода RM4SCC для сравнения. Без лишних слов, только практический код, который можно вставить в любой проект .NET.

## Требования

Прежде чем начать, убедитесь, что у вас есть:

* .NET 6.0 или новее (код также работает на .NET Framework 4.7+)
* Visual Studio 2022 (или любая другая IDE)
* NuGet‑пакет **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Папка на диске, куда можно записать сгенерированные PNG‑файлы (замените `YOUR_DIRECTORY` в коде).

Если что‑то из перечисленного вам незнакомо, не переживайте — установить NuGet‑пакет проще простого: правой кнопкой мыши по проекту → *Manage NuGet Packages* → поиск “Aspose.Barcode” → *Install*.

## Шаг 1 – Создание минимального консольного проекта

Сначала создадим небольшой консольный проект, чтобы запускать код штрих‑кода в изоляции.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Совет:** Держите метод `Main` лаконичным; делегируйте тяжёлую работу отдельным методам. Это упрощает тестирование и последующее чтение кода.

## Шаг 2 – **Создание штрих‑кода Planet** с высотой по умолчанию

Теперь действительно **создадим штрих‑код Planet**, используя высоту, вычисляемую автоматически. Параметр `XDimension` задаёт ширину одного модуля, а высота полос оставлена на усмотрение библиотеки.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Почему это работает?*  
`EncodeTypes.Planet` сообщает Aspose, что нам нужен символьный набор Planet, а `"123456"` — данные для кодирования. Не задавая `BarHeight`, движок сам подбирает высоту, соответствующую спецификации.

### Ожидаемый результат

Откройте `PlanetBarHeightAuto.png` — вы увидите чистый штрих‑код Planet высотой примерно 30 мм (точный размер зависит от DPI PNG‑писателя). Ручная настройка высоты не требуется.

## Шаг 3 – **Как сгенерировать штрих‑код Planet** с пользовательской высотой 100 пикселей

Если ваш дизайн требует фиксированной высоты, например 100 пикселей, просто задайте свойство `BarHeight` перед сохранением.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Обратите внимание на дополнительную строку `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Она заставляет штрих‑код растянуться (или сжаться) до ровно 100 пикселей, независимо от длины данных.

> **Внимание:** Слишком маленькая высота может привести к слиянию полос, и штрих‑код станет нечитаемым. Хорошее правило — держать высоту минимум в три раза больше X‑dimension.

## Шаг 4 – Генерация штрих‑кодов RM4SCC для сравнения

Иногда требуется сравнить Planet с другими почтовыми символьными наборами. RM4SCC — ещё один распространённый формат; код ниже повторяет шаги, использованные для Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Запуск полной программы сейчас создаст **четыре** PNG‑файла:

| Файл | Символика | Высота |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Вы можете открыть каждое изображение в любом просмотрщике, чтобы проверить размеры и визуальное качество.

## Шаг 5 – Проверка штрих‑кодов программно (необязательно)

Если хотите убедиться, что сгенерированные штрих‑коды сканируются, у Aspose есть также компонент **reader**. Ниже быстрый sanity‑check, подтверждающий, что закодированные данные совпадают с исходной строкой.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Выполнение `VerifyBarcodes()` выводит `Decoded value: 123456`, подтверждая, что пользовательская высота не нарушила кодирование.

## Распространённые ошибки и советы

| Проблема | Почему происходит | Как исправить |
|----------|-------------------|---------------|
| Штрих‑код выглядит слишком тонким | X‑dimension задан слишком низко (например, 1 пиксель) | Увеличьте `XDimension.Pixels` до 3‑5 для большинства принтеров |
| Изображение размыто | Сохранение как PNG с DPI по умолчанию (96) на экранах высокого разрешения | Используйте `ImageOptions` для установки `ResolutionX` и `ResolutionY` в 300 DPI |
| Высота не применяется | `BarHeight` установлен *после* вызова `Save()` | Всегда задавайте `BarHeight` **до** `Save()` |
| Неправильный путь к файлу | Папка `YOUR_DIRECTORY` не существует | Создайте её программно: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
