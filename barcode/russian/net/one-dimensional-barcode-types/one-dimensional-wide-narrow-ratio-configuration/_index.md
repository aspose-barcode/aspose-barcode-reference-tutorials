# Одномерная настройка отношения широких‑узких полос

Ищете, как **как генерировать штрих‑код** без усилий в ваших .NET‑приложениях? Aspose.BarCode for .NET делает генерацию штрих‑кодов в проектах Visual Studio простой и мощной. В этом руководстве мы пройдем процесс создания одномерных штрих‑кодов с пользовательским отношением широких‑узких полос, объясним, почему это отношение важно, и покажем, как настроить его для разных условий сканирования.

## Быстрые ответы
- **Что контролирует отношение широких‑узких полос?** Оно определяет относительную ширину «широких» полос по сравнению с «узкими» в 1D‑штрих‑коде.  
- **Какой тип штрих‑кода используется в примере?** Code 39 Extended, популярная символьная система для алфавитно‑цифровых данных.  
- **Можно ли изменить отношение во время выполнения?** Да — просто установите `gen.Parameters.Barcode.WideNarrowRatio` в нужное значение перед сохранением.  
- **Нужна ли лицензия для этой функции?** Отношение широких‑узких полос работает в бесплатной пробной версии; полная лицензия открывает все параметры рендеринга.  
- **Совместимо ли это с .NET Core?** Абсолютно — Aspose.BarCode поддерживает .NET Framework, .NET Core и .NET 5/6+.

## Что такое отношение широких‑узких полос?

В одномерных штрих‑кодах каждая полоса либо «широкая», либо «узкая». Отношение (например, 2 или 5) определяет, во сколько раз ширина широкой полосы превышает узкую. Регулировка этого отношения может улучшить читаемость на принтерах или сканерах с низким разрешением.

## Импорт пространств имён

Сначала необходимо добавить необходимые пространства имён в ваш проект, чтобы получить доступ к функциям Aspose.BarCode for .NET. Сделайте это, добавив следующие директивы using в начало вашего кода:

```csharp
using Aspose.BarCode;
using Aspose.BarCode.Generation;
```

## Шаг 1: Определите путь к каталогу

Сначала определите путь, куда вы хотите сохранять сгенерированные изображения штрих‑кодов. Это можно сделать с помощью следующего кода:

```csharp
string path = "Your Directory Path";
```

Замените `"Your Directory Path"` фактическим путём к каталогу, куда вы хотите сохранять изображения штрих‑кодов.

## Шаг 2: Создайте одномерный штрих‑код с настройкой отношения широких‑узких полос

Теперь создадим одномерный штрих‑код с настройкой отношения широких‑узких полос с помощью Aspose.BarCode for .NET. В этом примере мы используем тип кодирования Code39Extended и зададим данные `"ASPOSE"`:

```csharp
BarcodeGenerator gen = new BarcodeGenerator(EncodeTypes.Code39Extended, "ASPOSE");
```

Эта строка кода инициализирует генератор штрих‑кода с указанным типом кодирования и данными.

## Шаг 3: Установите отношение широких/узких полос

Отношение широких‑узких полос определяет соотношение между широкими и узкими элементами в штрих‑коде. На этом этапе мы установим отношение широких‑узких полос в **2**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 2;
```

Затем мы сохраним сгенерированное изображение штрих‑кода по указанному пути:

```csharp
gen.Save($"{path}WideNarrow2Code39.png", BarCodeImageFormat.Png);
```

## Шаг 4: Настройте отношение широких‑узких полос

Вы можете экспериментировать с различными отношениями широких‑узких полос, чтобы получить желаемый вид штрих‑кода. Например, если нужен более широкий штрих‑код, установите отношение в **5**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 5;
```

И сохраните изображение штрих‑кода:

```csharp
gen.Save($"{path}WideNarrow5Code39.png", BarCodeImageFormat.Png);
```

Теперь вы успешно создали одномерные штрих‑коды с различными отношениями широких‑узких полос с помощью Aspose.BarCode for .NET. Эта библиотека предоставляет гибкость для генерации штрих‑кодов, адаптированных к вашим конкретным требованиям.

## Распространённые проблемы и решения
- **Изображение не сохранено** — проверьте, что переменная `path` указывает на существующую папку и что у вашего приложения есть права на запись.  
- **Штрих‑код выглядит искажённым** — попробуйте уменьшить отношение (например, 2) для принтеров с низким разрешением; более высокие отношения могут вызывать артефакты печати.  
- **Неподдерживаемые символы** — Code 39 Extended поддерживает полный набор ASCII; убедитесь, что ваша строка данных не содержит запрещённых управляющих символов.
