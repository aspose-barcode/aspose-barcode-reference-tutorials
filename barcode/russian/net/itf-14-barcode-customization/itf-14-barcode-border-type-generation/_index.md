# Как изменить границу – генерация типа границы штрих‑кода ITF-14

В этом руководстве вы узнаете **как изменить границу** штрих‑кодов ITF-14 с помощью Aspose.BarCode для .NET. Независимо от того, создаёте ли вы систему упаковки‑маркировки или вам нужно соответствовать определённым стандартам печати, управление типом границы имеет решающее значение. Мы пройдём через полностью готовый пример, показывающий **генерацию штрих‑кода с использованием C#**, чтобы вы могли создавать штрих‑коды ITF-14 точно так, как вам нужно.

## Быстрые ответы
- **Что влияет “тип границы”?** Он определяет, будет ли штрих‑код нарисован без границы, простой полосой, внешней полосой, рамкой или рамкой с внешней полосой.  
- **Какая библиотека используется?** Aspose.BarCode для .NET.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; для продакшн‑использования требуется коммерческая лицензия.  
- **Можно ли запускать на .NET Core?** Да, API совместим с .NET Core, .NET 5+ и .NET 6+.  
- **Сколько строк кода?** Менее 20 строк для генерации всех пяти вариантов границы.

## Что означает “как изменить границу” в контексте штрих‑кодов ITF-14?
Изменение границы означает выбор одного из вариантов `ITF14BorderType` (`None`, `Bar`, `BarOut`, `Frame`, `FrameOut`). Каждый вариант меняет визуальное оформление штрих‑кода, что может быть важно для читаемости сканером и эстетических требований.

## Почему стоит использовать Aspose.BarCode для генерации штрих‑кодов на C#?
Aspose.BarCode предлагает богатый набор функций настройки — цвета, размеры, шрифты и типы границ, которые мы рассмотрим, — при этом API остаётся простым. Это делает её идеальной для разработчиков, которым нужно **генерировать изображения штрих‑кода ITF-14** быстро и надёжно.

## Предварительные требования

Прежде чем начать, убедитесь, что у вас есть:

1. **Aspose.BarCode для .NET** – скачайте её с [веб‑сайта](https://releases.aspose.com/barcode/net/).  
2. Среда разработки .NET (Visual Studio, Rider или VS Code).  
3. Базовое знакомство с синтаксисом **C#**.  
4. Действительный путь к папке, куда будут сохраняться сгенерированные PNG‑файлы – замените `"Your Directory Path"` в коде на свой путь.

## Импорт пространств имён

Сначала подключите необходимое пространство имён:

```csharp
using Aspose.BarCode;
```

## Пошаговое руководство

### Шаг 1: Создать экземпляр `BarcodeGenerator` (генерация штрих‑кода itf-14)

Инициализируем генератор с символьностью ITF‑14 и данными для кодирования:

```csharp
BarcodeGenerator gen = new BarcodeGenerator(EncodeTypes.ITF14, "12345678901231");
```

### Шаг 2: Установить X‑Dimension (ширина полосы)

X‑Dimension задаёт ширину каждой полосы штрих‑кода. Значение 2 пикселя хорошо подходит для большинства принтеров этикеток:

```csharp
gen.Parameters.Barcode.XDimension.Pixels = 2;
```

### Шаг 3: Генерировать штрих‑коды ITF‑14 с разными типами границы

Ниже представлены пять **примеров штрих‑кода ITF‑14**, демонстрирующих **как изменить границу**. Каждый фрагмент использует один и тот же экземпляр `BarcodeGenerator`, меняя только свойство `ItfBorderType`.

#### Тип границы ITF: None  

```csharp
gen.Parameters.Barcode.ITF.ItfBorderType = ITF14BorderType.None;
gen.Save($"{path}ITF14BorderNone.png", BarCodeImageFormat.Png);
```

#### Тип границы ITF: Bar  

```csharp
gen.Parameters.Barcode.ITF.ItfBorderType = ITF14BorderType.Bar;
gen.Save($"{path}ITF14BorderBar.png", BarCodeImageFormat.Png);
```

#### Тип границы ITF: BarOut  

```csharp
gen.Parameters.Barcode.ITF.ItfBorderType = ITF14BorderType.BarOut;
gen.Save($"{path}ITF14BorderBarOut.png", BarCodeImageFormat.Png);
```

#### Тип границы ITF: Frame  

```csharp
gen.Parameters.Barcode.ITF.ItfBorderType = ITF14BorderType.Frame;
gen.Save($"{path}ITF14BorderFrame.png", BarCodeImageFormat.Png);
```

#### Тип границы ITF: FrameOut  

```csharp
gen.Parameters.Barcode.ITF.ItfBorderType = ITF14BorderType.FrameOut;
gen.Save($"{path}ITF14BorderFrameOut.png", BarCodeImageFormat.Png);
```

Каждый вызов `Save` записывает PNG‑изображение в указанную директорию, предоставляя визуальную справку для каждого варианта границы.

## Распространённые проблемы и советы

- **Формат пути** – Убедитесь, что переменная `path` заканчивается обратным слешем (`\`) в Windows или прямым слешем (`/`) в Linux/macOS.  
- **Исключение лицензии** – При запуске кода без лицензии на сгенерированных изображениях появится небольшая водяная метка.  
- **Совместимость со сканером** – Некоторые сканеры игнорируют внешнюю границу; протестируйте с вашим оборудованием, чтобы решить, какой тип границы лучше подходит.  
- **Pro tip:** Вы можете цепочкой задавать несколько свойств (цвет, текст и т.д.) перед вызовом `Save`, чтобы создать полностью настроенный штрих‑код за один шаг.

## Часто задаваемые вопросы

### Для чего используется штрих‑код ITF-14?
Штрих‑коды ITF-14 в основном применяются для упаковки и маркировки товаров в розничной торговле. Они кодируют информацию, такую как GTIN (Global Trade Item Number) продукта, и обычно находятся на коробках и поддонах.

### Можно ли настроить внешний вид штрих‑кодов ITF-14 с помощью Aspose.BarCode?
Да, Aspose.BarCode предоставляет обширные возможности настройки, включая изменение типа границы штрих‑кода, цвета и многих других визуальных аспектов.

### Совместима ли Aspose.BarCode с другими фреймворками .NET?
Да, Aspose.BarCode для .NET совместим с различными фреймворками .NET, включая .NET Core и .NET Standard, а также с традиционным .NET Framework.
