## Часто задаваемые вопросы

**Q: Как решить, какой режим ECC использовать?**  
A: Выбирайте ECC 000‑140 для небольших наборов данных с ограниченной коррекцией ошибок, или ECC 200 для больших объёмов данных и более высокой надёжности. Макро‑режим добавляет дополнительные уровни данных.

**Q: Можно ли встроить пользовательский текст в штрих‑код DataMatrix?**  
A: Да, используйте свойство `CodeText` для установки расширенного текста кода, затем при необходимости настройте режим кодирования.

**Q: Есть ли способ автоматически выбирать лучший режим кодирования?**  
A: Установите `EncodeMode` в `Auto`, и Aspose.BarCode оценит входные данные и выберет оптимальный режим (ASCII, C40, Bytes и т.д.).

**Q: Какие соображения по производительности при работе с большими партиями штрих‑кодов?**  
A: Переиспользуйте экземпляр `BarCodeGenerator` и включайте многопоточность, где это возможно. Кроме того, генерируйте изображения в подходящем формате (PNG для без потерь, JPEG для меньшего размера).

**Q: Поддерживает ли Aspose.BarCode .NET Core и .NET 5/6?**  
A: Абсолютно — библиотека полностью совместима с .NET Framework, .NET Core и последними версиями .NET.
