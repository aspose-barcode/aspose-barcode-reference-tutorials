# Aspose Barcode Java – Создание изображения штрих‑кода с символами начала/конца

## Введение

В этом полном руководстве вы **создадите файлы изображений штрих‑кода java** с помощью **Aspose Barcode Java** и узнаете **как задавать символы** для штрих‑кодов Codabar. Независимо от того, разрабатываете ли вы систему точек продаж, логистическое приложение или любое решение, требующее надёжного создания штрих‑кодов, настройка символов начала и конца даёт вам полный контроль над форматом штрих‑кода. Мы пройдём каждый шаг, объясним, почему важна каждая настройка, и покажем, как получить готовое PNG‑изображение — без водяного знака пробной версии.

## Быстрые ответы
- **Какая библиотека создаёт изображения штрих‑кодов в Java?** Aspose.BarCode for Java.  
- **Можно ли настроить символы начала/конца?** Да, с помощью `setCodabarStartSymbol` и `setCodabarStopSymbol`.  
- **Какой тип штрих‑кода используется в этом примере?** CODABAR.  
- **Нужна ли лицензия для продакшн‑использования?** Для использования без пробной версии требуется коммерческая лицензия.  
- **В каком формате генерируется вывод?** PNG‑изображение, сохраняемое на диск.

## Что такое Aspose Barcode Java?

Aspose Barcode Java — мощная библиотека без внешних зависимостей, позволяющая программно генерировать широкий спектр символьных систем штрих‑кодов. Она скрывает низкоуровневую логику кодирования, так что вы можете сосредоточиться на бизнес‑правилах, гарантируя соответствие вывода отраслевым стандартам.

## Почему стоит использовать Aspose Barcode Java для генерации штрих‑кодов без водяного знака?

- **Отсутствие водяного знака в продакшн‑режиме** — после применения действующей лицензии изображения чистые.  
- **Широкая поддержка символьных систем** — от классических 1D‑кодов, таких как CODABAR, до 2D‑кодов, например QR и DataMatrix.  
- **Тонкая настройка** — можно задавать символы начала/конца, цвета, размеры и даже генерировать изображения напрямую в потоки для веб‑приложений.

## Импорт пакетов

В вашем Java‑проекте импортируйте необходимые классы для работы с Aspose.BarCode:

```java
// Import Aspose.BarCode classes
import com.aspose.barcode.CodabarSymbol;
import com.aspose.barcode.generation.BarcodeGenerator;
```

## Пошаговое руководство

### Шаг 1: Определите каталог документов

```java
// The path to the resource directory.
String dataDir = "Your Document Directory";
```

Замените `"Your Document Directory"` на абсолютный или относительный путь, где вы хотите сохранить изображение штрих‑кода. Не забудьте завершить путь соответствующим разделителем файлов (`/` или `\`) или используйте `Paths.get` для платформенно‑независимого формирования пути.

### Шаг 2: Создайте экземпляр Barcode Generator

```java
// Create instance of BarcodeGenerator, specify codetext and symbology in the constructor
BarcodeGenerator generator = new BarcodeGenerator(com.aspose.barcode.EncodeTypes.CODABAR, "12345678");
```

Здесь мы указываем Aspose.BarCode использовать символьную систему **CODABAR** и строку данных `"12345678"`.

### Шаг 3: Установите символ начала Codabar

```java
// Set the Codabar start symbol to A
generator.getParameters().getBarcode().getCodabar().setCodabarStartSymbol(CodabarSymbol.A);
```

Метод `setCodabarStartSymbol` позволяет **задать символы штрих‑кода** для начального символа. В данном случае выбираем `A`.

### Шаг 4: Установите символ окончания Codabar

```java
// Set the Codabar stop symbol to D
generator.getParameters().getBarcode().getCodabar().setCodabarStopSymbol(CodabarSymbol.D);
```

Аналогично, `setCodabarStopSymbol` задаёт символ окончания. Использование `D` завершает формат CODABAR, требуемый многими устаревшими системами.

### Шаг 5: Сохраните сгенерированное изображение

```java
// Save the image to disk in PNG format
generator.save(dataDir + "startAndStopSymbols.png");
```

Вызов `save` записывает штрих‑код в PNG‑файл **startAndStopSymbols.png** в каталог, указанный ранее.

## Распространённые ошибки и советы

- **Неправильный путь к каталогу** — убедитесь, что `dataDir` заканчивается разделителем файлов (`/` или `\`) или объединяйте путь через `Paths.get`.  
- **Неподдерживаемые символы начала/конца** — CODABAR поддерживает только `A`, `B`, `C`, `D` в качестве символов начала/конца. Любое другое значение вызовет исключение.  
- **Лицензия не применена** — в пробном режиме сгенерированное изображение может содержать водяной знак. Примените лицензию перед развертыванием в продакшн, чтобы получить **генерацию штрих‑кода без водяного знака**.
