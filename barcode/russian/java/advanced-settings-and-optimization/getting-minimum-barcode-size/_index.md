# Как создать минимальный штрих‑код в Java с Aspose.BarCode

## Введение

Если вам нужно **создать минимальный штрих‑код** для плотных UI‑макетов, этикеток, готовых к печати, или любой ситуации, где каждый миллиметр имеет значение, вы попали по адресу. В этом **barcode generator tutorial Java** мы пройдем точные шаги, необходимые для уменьшения штрих‑кода до минимального возможного размера при сохранении его читаемости, используя Aspose.BarCode для Java.

## Быстрые ответы
- **Что означает «минимальный штрих‑код»?** Это наименьшие размеры изображения, которые всё ещё удовлетворяют требованиям читаемости символьного набора.  
- **Какой класс генерирует штрих‑код?** `BarcodeGenerator` из библиотеки Aspose.BarCode.  
- **Нужна ли лицензия для этого примера?** Бесплатная пробная версия подходит для разработки; для продакшн‑использования требуется коммерческая лицензия.  
- **Можно ли изменить размер после отключения AutoSize?** Да — вы задаёте явные значения ширины/высоты в миллиметрах.  
- **Подходит ли этот подход для всех типов штрих‑кодов?** Большинство 1‑D символьных наборов (например, CODE_128, CODE_39) поддерживают ручное задание размеров; для 2‑D кодов проверьте документацию.

## Что означает «создать минимальный штрих‑код»?
Создание минимального штрих‑кода означает настройку генератора так, чтобы он **не** увеличивал изображение автоматически. Вместо этого вы указываете точные необходимые размеры, позволяя разместить штрих‑код в узких пространствах без лишних пустых областей.

## Почему использовать такой barcode generator tutorial Java?
- **Эффективные по пространству дизайны** — идеально подходят для мобильных экранов, чеков или компактных принтеров этикеток.  
- **Полный контроль** — вы задаёте точный размер в пикселях или миллиметрах.  
- **Последовательные результаты** — тот же код работает на JVM под Windows, Linux и macOS.

## Предварительные требования

Прежде чем перейти к коду, убедитесь, что у вас есть:

1. **Java Development Kit (JDK)** — любая современная версия (рекомендовано 8+).  
2. **Aspose.BarCode for Java** — скачайте последнюю библиотеку с официального сайта [here](https://releases.aspose.com/barcode/java/).

Теперь приступим к кодированию.

## Импорт пространств имён

В вашем Java‑файле импортируйте необходимые классы Aspose:

```java
import com.aspose.barcode.generation.AutoSizeMode;
import com.aspose.barcode.generation.BarcodeGenerator;
```

## Пошаговое руководство по созданию минимального штрих‑кода

### Шаг 1: Настройка генератора штрих‑кода
Создайте экземпляр `BarcodeGenerator`, выберите символьный набор (в примере CODE_128) и укажите данные для кодирования.

```java
BarcodeGenerator bb = new BarcodeGenerator(EncodeTypes.CODE_128, "1234567");
```

### Шаг 2: Отключение AutoSizeMode
По умолчанию Aspose.BarCode расширяет изображение, чтобы вместить штрих‑код. Отключите это поведение, чтобы задать собственные размеры.

```java
bb.getParameters().setAutoSizeMode(AutoSizeMode.NONE);
```

### Шаг 3: Определение минимальной ширины и высоты изображения
Укажите минимальную ширину и высоту, которые всё ещё позволяют считывать штрих‑код. Здесь мы используем 1 mm для обеих измерений, но при необходимости можете изменить их.

```java
bb.getParameters().getImageWidth().setMillimeters(1);
bb.getParameters().getImageHeight().setMillimeters(1);
```

> **Pro tip:** Используйте свойства `getImageWidth()` и `getImageHeight()`, чтобы экспериментировать с различными размерами, пока сканер надёжно не считывает код.

### Шаг 4: Сохранение изображения штрих‑кода
Сгенерируйте bitmap и запишите его в PNG‑файл. Замените `dataDir` на путь, где вы хотите сохранить изображение.

```java
javax.imageio.ImageIO.write(bb.generateBarCodeImage(), "PNG", new java.io.File(dataDir + "minimumresult.png"));
```

Повторите вышеуказанные шаги для каждого штрих‑кода, который нужно сгенерировать с минимальным размером.

| Штрих‑код становится нечитаемым | Ширина/высота слишком малы для выбранного символьного набора | Увеличьте значения миллиметров постепенно (например, 1.2 mm) и протестируйте сканером. |
