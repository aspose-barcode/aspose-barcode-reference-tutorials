# Как отключить проверку контрольной суммы в Java

В современных приложениях для инвентаризации и логистики **как отключить контрольную сумму** может стать ключом к чтению устаревших штрих‑кодов, в которых отсутствует цифра контрольной суммы. Aspose.BarCode for Java позволяет легко отключить проверку контрольной суммы, одновременно извлекая закодированные данные. Этот учебник проведёт вас через весь процесс — от настройки проекта до чтения штрих‑кода ONE‑CODE без проверки контрольной суммы.

## Быстрые ответы
- **Что делает отключение контрольной суммы?** Инструктирует считыватель игнорировать поле контрольной суммы, позволяя обрабатывать штрих‑коды без действительной контрольной суммы.  
- **Когда следует отключать контрольную сумму?** При работе с устаревшими системами или нестандартными штрих‑кодами, в которых контрольная сумма опущена или повреждена.  
- **Какой класс управляет проверкой контрольной суммы?** `BarCodeReader.setChecksumValidation(ChecksumValidation)`  
- **Безопасно ли отключать контрольную сумму?** Только если вы доверяете источнику штрих‑кода; в противном случае проверка помогает обнаружить ошибки.  
- **Влияет ли это на другие типы штрих‑кодов?** Настройка применяется только к текущему экземпляру `BarCodeReader`.

## Что такое проверка контрольной суммы?
Проверка контрольной суммы — это проверка целостности данных, которая вычисляет небольшое значение из содержимого штрих‑кода и сравнивает его с встроенной контрольной суммой. Если они не совпадают, штрих‑код считается нечитаемым. Отключение этой проверки заставляет Aspose.BarCode пропустить сравнение.

## Почему отключать контрольную сумму с Aspose.BarCode?
- **Поддержка наследия:** Старые сканеры часто генерировали штрих‑коды без контрольных сумм.  
- **Производительность:** Пропуск вычисления может ускорить массовое чтение.  
- **Гибкость:** Вы можете решать для каждого экземпляра считывателя, применять проверку или нет.

## Импорт пакетов
В вашем Java‑проекте импортируйте необходимые классы Aspose.BarCode для работы с распознаванием штрих‑кодов.

```java
// Import Aspose.BarCode classes
import com.aspose.barcode.barcoderecognition.BarCodeReader;
import com.aspose.barcode.barcoderecognition.BarCodeResult;
import com.aspose.barcode.barcoderecognition.ChecksumValidation;
import com.aspose.barcode.barcoderecognition.DecodeType;
```

## Пошаговое руководство

### Шаг 1: Настройте проект
Создайте новый Java‑проект (в любой IDE) и добавьте JAR‑файл Aspose.BarCode в classpath проекта.

### Шаг 2: Инициализируйте `BarCodeReader`
Загрузите изображение, содержащее штрих‑код ONE‑CODE, который нужно прочитать.

```java
String dataDir = "Your Document Directory";
BarCodeReader reader = new BarCodeReader(dataDir + "onecode.png", DecodeType.ONE_CODE);
```

### Шаг 3: Как отключить контрольную сумму
Укажите считывателю игнорировать проверку контрольной суммы, установив свойство в `OFF`.

```java
reader.setChecksumValidation(ChecksumValidation.OFF);
```

### Шаг 4: Чтение штрих‑кодов
Пройдитесь по результатам и выведите декодированный текст и тип символьного набора.

```java
for (BarCodeResult result : reader.readBarCodes()) {
    System.out.println("CodeText: " + result.getCodeText());
    System.out.println("Symbology type: " + result.getCodeType());
}
```

**Результат:** Текст штрих‑кода отображается, даже если исходное изображение не содержит действительной контрольной суммы.

## Распространённые проблемы и советы
- **Неправильный путь к файлу:** Убедитесь, что `dataDir` указывает на правильную папку; для тестов используйте абсолютные пути.  
- **Неподдерживаемый тип штрих‑кода:** Убедитесь, что `DecodeType` соответствует штрих‑коду, который вы читаете.  
- **Производительность:** Отключение контрольной суммы при обработке больших пакетов может повысить пропускную способность, но всегда включайте её для критически важных данных.
