# Извлечение текста из штрих‑кода: PDF417 с китайскими символами в Java

## Введение

Интеграция распознавания штрих‑кодов в Java‑приложения — ценное умение, особенно когда нужно **извлекать текст из изображений штрих‑кода**, содержащих многоязычные данные. В этом руководстве мы покажем, как с помощью Aspose.BarCode for Java распознавать штрих‑коды PDF417 с китайскими символами. К концу вы точно знаете, как читать данные штрих‑кода и декодировать их в читаемый текст.

## Быстрые ответы
- **Какая библиотека поддерживает PDF417 с китайскими символами?** Aspose.BarCode for Java.  
- **Какой набор символов нужен для декодирования китайского?** MS936 (GBK).  
- **Нужна ли лицензия для продакшн‑использования?** Да, требуется коммерческая лицензия.  
- **Можно ли запускать на любой версии Java?** Работает с Java 8 и новее.  
- **Доступна ли бесплатная пробная версия?** Конечно — скачайте её с сайта Aspose.

## Что значит «извлекать текст из штрих‑кода»?

Извлечение текста из штрих‑кода означает преобразование закодированных данных обратно в их исходную человекочитаемую форму. Для штрих‑кодов PDF417, содержащих китайские символы, это также подразумевает выбор правильной кодировки символов, чтобы байты соответствовали нужным глифам.

## Почему стоит использовать Aspose.BarCode for Java?

Aspose.BarCode предоставляет надёжную поддержку широкого спектра символьных систем штрих‑кодов, включая PDF417, и обрабатывает сложные наборы символов «из коробки». Библиотека абстрагирует низкоуровневую обработку изображений, позволяя сосредоточиться на бизнес‑логике, а не на тонкостях декодирования.

## Предварительные требования

Прежде чем приступить, убедитесь, что у вас есть:

1. **Java Development Kit (JDK)** — рекомендуется последняя версия.  
2. **Aspose.BarCode for Java** — скачайте её [здесь](https://releases.aspose.com/barcode/java/).  
3. **Изображение штрих‑кода PDF417**, содержащее китайские символы (например, `barcode.png`).

## Импорт пакетов

В вашем Java‑проекте импортируйте необходимые классы для работы с Aspose.BarCode:

```java
import java.nio.ByteBuffer;
import java.nio.charset.Charset;

import com.aspose.barcode.barcoderecognition.BarCodeReader;
import com.aspose.barcode.barcoderecognition.BarCodeResult;
import com.aspose.barcode.barcoderecognition.DecodeType;
```

## Шаг 1: Установите каталог документов

Укажите папку, где находится изображение вашего штрих‑кода:

```java
String dataDir = "Your Document Directory";
```

Замените `"Your Document Directory"` на фактический путь на вашем компьютере.

## Шаг 2: Загрузите изображение штрих‑кода

Создайте экземпляр `BarCodeReader`, указывающий на PNG‑файл и задающий поиск символьной системы PDF417:

```java
BarCodeReader reader = new BarCodeReader(dataDir + "barcode.png", DecodeType.PDF_417);
```

## Шаг 3: Чтение штрих‑кода

Пройдитесь по результатам обнаружения, извлеките массив необработанных байтов и декодируйте его с помощью кодировки GBK (MS936):

```java
for (BarCodeResult result : reader.readBarCodes()) {
    byte[] bytes = result.getCodeBytes();
    ByteBuffer bytebuf = ByteBuffer.wrap(bytes);
    System.out.println(Charset.forName("MS936").decode(bytebuf).toString());
}
```

Этот цикл **извлекает текст из штрих‑кода** и выводит декодированные китайские символы в консоль.

## Как читать штрих‑код с PDF417?

Приведённый выше код демонстрирует **как читать данные штрих‑кода** шаг за шагом:

1. **Инициализировать** считыватель с правильным `DecodeType`.  
2. **Итерировать** каждый `BarCodeResult`, возвращённый считывателем.  
3. **Преобразовать** необработанные байты, используя соответствующую кодировку (`MS936` для китайского).  

Если ваш штрих‑код содержит другие языки, просто замените кодировку на нужную для ваших данных.

## Распространённые проблемы и решения

| Проблема | Решение |
|----------|---------|
| Иероглифы отображаются как «кракозябры» после декодирования | Убедитесь, что используете правильную кодировку (`MS936` для GBK). |
| Штрих‑код не обнаружен | Проверьте качество изображения и отсутствие вращения; при необходимости предобработайте изображение. |
| Возвращается несколько результатов | PDF417 может хранить несколько сегментов; итерируйте все результаты, как показано. |

**В: Что делать, если моё изображение штрих‑кода в формате JPEG?**  
О: `BarCodeReader` работает с JPEG, PNG, BMP и другими распространёнными форматами изображений — просто измените расширение файла.

**В: Можно ли декодировать штрих‑коды из потока, а не из файла?**  
О: Да, в конструктор `BarCodeReader` можно передать `InputStream` для декодирования «на лету».
