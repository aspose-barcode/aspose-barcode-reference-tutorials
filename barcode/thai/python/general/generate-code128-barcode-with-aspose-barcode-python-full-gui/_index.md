# สร้างบาร์โค้ด Code128 ด้วย Aspose.Barcode Python – คู่มือเต็ม

เคยต้อง **สร้างบาร์โค้ด Code128** ในโปรเจกต์ Python แต่ไม่แน่ใจว่าจะฝังอักขระที่ไม่ใช่ ASCII อย่างไรหรือไม่? คุณไม่ได้อยู่คนเดียว นักพัฒนาจำนวนมากเจออุปสรรคเมื่อพยายามผสานข้อมูลตัวเลขกับข้อความ Unicode—โดยเฉพาะภาษารัสเซียหรือจีน—ภายในบาร์โค้ดเดียว ข่าวดีคือ Aspose.Barcode สำหรับ Python ทำให้เรื่องนี้ง่ายดาย และด้วยการใช้ *barcode with ECI* อย่างถูกต้อง คุณสามารถทำให้ข้อมูลทั้งหมดอ่านได้โดยสแกนเนอร์ทั่วโลก

ในบทแนะนำนี้เราจะเดินผ่านตัวอย่างที่ทำงานได้เต็มรูปแบบซึ่งแสดงอย่างชัดเจนว่า **สร้างบาร์โค้ด Code128** อย่างไรโดยใช้ไลบรารี Aspose.Barcode Python, วิธีสร้างสตริง codetext แบบขยายที่รวมส่วน ECI, และเหตุผลที่แต่ละบรรทัดสำคัญ หลังจากนี้คุณจะมีสคริปต์พร้อมรัน, เข้าใจแนวคิดพื้นฐาน, และได้รับเคล็ดลับมืออาชีพเพื่อหลีกเลี่ยงข้อผิดพลาดทั่วไป

## สิ่งที่คุณจะได้เรียนรู้

- ตั้งค่า **Aspose.Barcode Python** ในสภาพแวดล้อมของคุณ  
- สร้างสตริง codetext แบบขยายที่ผสมตัวเลขธรรมดากับข้อความ UTF‑8 ผ่านส่วน ECI  
- กำหนดค่า `BarCodeGenerator` เพื่อสร้าง **Code128 barcode**  
- บันทึกภาพและตรวจสอบผลลัพธ์  
- การจัดการกรณีขอบสำหรับชุดอักขระและสัญลักษณ์บาร์โค้ดที่ต่างกัน  

ไม่ต้องอ้างอิงเอกสารภายนอก—ทุกอย่างที่คุณต้องการอยู่ที่นี่

## ข้อกำหนดเบื้องต้น

1. **Python 3.8+** ติดตั้งแล้ว (ไลบรารีรองรับ 3.6 ขึ้นไป)  
2. ใบอนุญาต **Aspose.Barcode for Python via .NET** ที่ถูกต้องหรือคุณยอมรับลายน้ำการประเมินผล  
3. เข้าถึงเทอร์มินัลหรือ IDE ที่คุณสามารถติดตั้งแพ็กเกจ (`pip`)  
4. โฟลเดอร์ที่ PNG ที่สร้างขึ้นจะถูกบันทึก (แทนที่ `YOUR_DIRECTORY` ในโค้ด)  

หากรายการใดฟังดูแปลกใหม่ ไม่ต้องกังวล—การติดตั้งแพ็กเกจทำได้ด้วยคำสั่งเดียวและส่วนที่เหลือเป็น Python ธรรมดา

## ขั้นตอนที่ 1: ติดตั้ง Aspose.Barcode สำหรับ Python

เปิดเทอร์มินัลของคุณและรัน:

```bash
pip install aspose-barcode
```

> **เคล็ดลับมืออาชีพ:** ใช้ virtual environment (`python -m venv venv`) เพื่อแยกการพึ่งพาออกจากกัน ซึ่งยังช่วยป้องกันการชนกันของเวอร์ชันกับโปรเจกต์อื่น  

แพ็กเกจจะดึง runtime ของ .NET มาใช้งานภายใต้พื้นฐาน ดังนั้นการ import ครั้งแรกอาจใช้เวลาสักครู่ขณะโหลดไลบรารีเนทีฟ

## ขั้นตอนที่ 2: สร้าง Extended Codetext Builder

หัวใจของการสนับสนุน **barcode with ECI** อยู่ที่ `ExtCodetextBuilder` ซึ่งช่วยให้คุณต่อส่วนธรรมดากับส่วนที่เข้ารหัสด้วย ECI ได้อย่างถูกต้อง ทำให้สแกนเนอร์ตีความข้อความ Unicode ได้อย่างแม่นยำ

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**ทำไมต้อง clear?**  
หากคุณใช้ builder ซ้ำในลูป ข้อมูลที่ค้างอยู่อาจทำให้บาร์โค้ดถัดไปเสียหาย การเรียก `clear()` จะทำให้เริ่มต้นใหม่ทุกครั้ง

## ขั้นตอนที่ 3: เพิ่มส่วนตัวเลขธรรมดา

บาร์โค้ด Code128 ส่วนใหญ่เริ่มต้นด้วยตัวระบุตัวเลข—เช่นหมายเลขออเดอร์หรือรหัสสินค้า การเพิ่มเป็นส่วน *plain* ทำให้การเข้ารหัสง่ายและกระชับ

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

เมธอดจะเลือกชุด Code128 ที่เหมาะสมที่สุด (A, B หรือ C) สำหรับตัวเลขโดยอัตโนมัติ ส่งผลให้บาร์โค้ดมีขนาดเล็กที่สุดที่เป็นไปได้

## ขั้นตอนที่ 4: แทรกส่วน ECI พร้อมข้อความรัสเซีย

ตอนนี้มาส่วนที่ท้าทาย: ฝังอักขระรัสเซีย (`Пример текста`) หากไม่มีเครื่องหมาย ECI สแกนเนอร์ส่วนใหญ่จะตีความไบต์เหล่านั้นผิดพลาดและแสดงผลเป็นข้อความเสียหาย

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**เกิดอะไรขึ้น?**  
- `26` บอกบาร์โค้ดว่าบายต์ต่อไปใช้การเข้ารหัส UTF‑8  
- ไลบรารีจัดการการแปลงให้โดยอัตโนมัติ คุณเพียงแค่ส่งสตริง Unicode  
- เมื่อสแกนเนอร์อ่านบาร์โค้ด มันจะเห็น ECI, สลับไปใช้ UTF‑8, และแสดงข้อความรัสเซียได้อย่างถูกต้อง  

> **คำถามทั่วไป:** *ถ้าต้องการภาษาอื่นล่ะ?*  
> ใช้ค่า ECI ที่เหมาะสม (เช่น 27 สำหรับ ISO‑8859‑1, 28 สำหรับ ISO‑8859‑2) ตารางอ้างอิงสั้น ๆ มีในเอกสาร Aspose แต่รูปแบบการใช้งานยังคงเหมือนเดิม

## ขั้นตอนที่ 5: ดึง Extended Codetext เต็มรูปแบบ

เมื่อเพิ่มส่วนทั้งสองแล้ว เราจะดึงสตริงที่รวมกัน นี่คือค่าที่คุณจะกำหนดให้กับ `code_text` ของ generator

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

ผลลัพธ์ที่พิมพ์ออกมาจะคล้ายกับ:

```
Extended codetext: 1234567890[ECI26]Пример текста
```

โทเค็น `[ECI26]` ไม่ปรากฏในภาพสุดท้าย แต่มีความสำคัญต่อการถอดรหัสที่ถูกต้อง

## ขั้นตอนที่ 6: กำหนดค่า BarCodeGenerator สำหรับ Code128

ตอนนี้เราจะเชื่อมทุกอย่างเข้าด้วยกัน `BarCodeGenerator` คือเครื่องมือหลักที่เรนเดอร์ภาพ

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**ทำไมต้องตั้งค่า `x_dimension`?**  
ความกว้างโมดูลที่เล็กลงทำให้บาร์โค้ดกระชับขึ้น—เหมาะกับพื้นที่จำกัด ปรับค่าให้สอดคล้องกับ DPI ของเครื่องพิมพ์ของคุณ

## ขั้นตอนที่ 7: บันทึกภาพบาร์โค้ด

สุดท้ายให้เขียน PNG ลงดิสก์ คุณยังสามารถส่งออกเป็น JPEG, BMP หรือแม้แต่ SVG โดยเปลี่ยนนามสกุลไฟล์

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

เปิดภาพด้วยโปรแกรมดูใดก็ได้ คุณควรเห็น Code128 ที่สะอาดและเมื่อสแกนด้วยเครื่องอ่านสมัยใหม่ จะคืนค่า `1234567890Пример текста`

## สคริปต์เต็ม – คัดลอกและรัน

ด้านล่างเป็นสคริปต์สมบูรณ์ที่ทำงานได้เอง คัดลอกไปไฟล์ชื่อ `generate_code128.py` ปรับ `YOUR_DIRECTORY` แล้วรันด้วย `python generate_code128.py`

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**ผลลัพธ์ที่คาดหวัง** (คอนโซล):

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

เปิด `extended_code.png` แล้วคุณจะเห็น Code128 ที่คมชัดพร้อมใช้งานในผลิตภัณฑ์

## การเจาะลึก: วิธีการทำงานของกลไก ECI

ECI (Extended Channel Interpretation) เป็นส่วนหนึ่งของมาตรฐาน QR และบาร์โค้ดที่บ่งบอกการเปลี่ยนแปลงการเข้ารหัสอักขระระหว่างสตรีม ในบริบทของ Code128:

1. บาร์โค้ดเริ่มต้นในโหมด ISO‑8859‑1 เริ่มต้น  
2. เมื่อสแกนเนอร์พบรหัสควบคุม ECI (ค่า 26 ในตัวอย่างของเรา) มันจะสลับไปใช้ UTF‑8  
3. ไบต์ต่อมาทั้งหมดจะถูกตีความเป็น UTF‑8 จนกว่าจะเจอ ECI อีกครั้งหรือจนจบข้อมูล  

เพราะเราเรียก `add_eci_codetext(26, ...)` Aspose จะใส่ตัวอักษรควบคุมที่ถูกต้องโดยอัตโนมัติ หากคุณละเว้น ECI ข้อความรัสเซียจะถูกเข้ารหัสด้วยหน้าโค้ดเริ่มต้นซึ่งมักทำให้สแกนเนอร์แสดงผลเป็นอักขระเสียหาย

## การจัดการกรณีขอบ

### 1. หลายส่วน ECI

หากต้องฝังข้อความญี่ปุ่นและอาหรับพร้อมกัน คุณสามารถเรียก `add_eci_codetext` หลายครั้งด้วยค่า ECI ที่ต่างกัน Builder จะต่อสตริงตามลำดับที่คุณเพิ่ม

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. ขีดจำกัดความยาว

Code128 สามารถเข้ารหัสได้สูงสุด 2 023 ตัวอักษร แต่ขีดจำกัดจริงมักต่ำกว่าเนื่องจากหน่วยความจำของสแกนเนอร์ หากเกินจะทำให้ generator โยน `ArgumentException` เพื่อความปลอดภัย ควรเก็บความยาวรวมไว้ต่ำกว่า 1 000 ตัวอักษรเมื่อผสมส่วน ECI

### 3. ความเข้ากันได้กับสแกนเนอร์รุ่นเก่า

สแกนเนอร์รุ่นเก่าบางรุ่นอาจละเลยรหัสควบคุม ECI ในสภาพแวดล้อมเช่นนี้ คุณอาจต้องสร้างบาร์โค้ดแยกสองอัน—หนึ่งสำหรับส่วนตัวเลข อีกหนึ่งสำหรับข้อความ Unicode—หรือใช้สัญลักษณ์อื่นเช่น QR ที่รองรับ ECI อย่างกว้างขวาง

## เคล็ดลับมืออาชีพสำหรับการใช้งานในผลิตภัณฑ์

- **ลงทะเบียนลิขสิทธิ์ตั้งแต่ต้น:** เวอร์ชันประเมินผลจะเพิ่มลายน้ำเล็กน้อย ใส่ไฟล์ลิขสิทธิ์ของคุณก่อนการเรียก `BarCodeGenerator` ครั้งแรกเพื่อหลีกเลี่ยง  
- **การประมวลผลเป็นชุด:** เมื่อสร้างบาร์โค้ดหลายพันรายการ ใช้ `ExtCodetextBuilder` ตัวเดียว—แค่เรียก `clear()` ทุกครั้งที่วนลูป  
- **ความปลอดภัยของเธรด:** แต่ละเธรดควรมี `BarCodeGenerator` ของตนเอง; วัตถุเหล่านี้ไม่ปลอดภัยต่อเธรด  
- **ประสิทธิภาพ:** ตั้งค่า `x_dimension` ต่ำเกินไปอาจทำให้เวลาเรนเดอร์เพิ่มขึ้น ทดสอบประสิทธิภาพกับฮาร์ดแวร์เป้าหมายของคุณหากอยู่ในสถานการณ์ที่ต้องประมวลผลสูง  
- **การทดสอบ:** สแกนบาร์โค้ดที่สร้างขึ้นด้วยเครื่องอ่านอย่างน้อยสองตัว (แอปมือถือและสแกนเนอร์ฮาร์ดแวร์) เพื่อยืนยันการจัดการ ECI
