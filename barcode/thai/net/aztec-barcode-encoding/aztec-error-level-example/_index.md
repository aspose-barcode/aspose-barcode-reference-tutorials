# วิธีสร้างบาร์โค้ด Aztec พร้อมการแก้ไขข้อผิดพลาดใน .NET

ในบทแนะนำแบบขั้นตอนนี้ คุณจะได้เรียนรู้วิธี **สร้างบาร์โค้ด Aztec** ที่รวมระดับการแก้ไขข้อผิดพลาดที่แตกต่างกันโดยใช้ไลบรารี Aspose.BarCode สำหรับ .NET ไม่ว่าคุณจะต้องการบาร์โค้ดที่ทนทานสำหรับบรรจุภัณฑ์ การออกบัตร หรือการสแกนบนมือถือ การควบคุมระดับข้อผิดพลาดจะช่วยให้คุณสมดุลระหว่างความจุข้อมูลและความทนทานต่อความเสียหาย เราจะเดินผ่านแต่ละตัวเลือกการกำหนดค่า แสดงโค้ดที่คุณต้องใช้อย่างแม่นยำ และอธิบายว่าทำไมการตั้งค่าแต่ละอย่างจึงสำคัญ

## คำตอบอย่างรวดเร็ว

- **ฉันสามารถตั้งระดับการแก้ไขข้อผิดพลาดแบบกำหนดเองได้หรือไม่?** ใช่ – จำนวนเต็มใดก็ได้ตั้งแต่ 0 % ถึง 100 %

- **รูปแบบไฟล์ที่รองรับ?** PNG, JPEG, BMP, GIF, และอื่น ๆ  

## การสร้างบาร์โค้ด Aztec พร้อมการแก้ไขข้อผิดพลาดคืออะไร?
บาร์โค้ด Aztec เป็นรหัสเมทริกซ์สองมิติที่สามารถเก็บข้อมูลจำนวนมากในรูปแบบสี่เหลี่ยมจัตุรัสที่กะทัดรัด การแก้ไขข้อผิดพลาดจะเพิ่มข้อมูลซ้ำเพื่อให้บาร์โค้ดยังคงอ่านได้แม้บางส่วนจะเสียหายหรือถูกบัง การปรับระดับการแก้ไขข้อผิดพลาดทำให้คุณเลือกได้ระหว่างความจุข้อมูลสูง (ระดับข้อผิดพลาดต่ำ) หรือความทนทานสูง (ระดับข้อผิดพลาดสูง)

## ทำไมต้องใช้ Aspose.BarCode สำหรับ .NET?
Aspose.BarCode มี API แบบ fluent ที่ซ่อนรายละเอียดการเข้ารหัสระดับล่าง ทำให้คุณมุ่งเน้นที่ตรรกะธุรกิจได้ง่ายขึ้น รองรับมาตรฐานบาร์โค้ดหลากหลาย มีการปรับแต่งที่ครอบคลุม (ขนาด, สี, ระยะขอบ) และทำงานได้บน .NET Framework, .NET Core, และ .NET 5/6+ ทำให้เป็นตัวเลือกที่เหมาะสำหรับแอปพลิเคชันระดับองค์กรที่ต้องการความน่าเชื่อถือและความยืดหยุ่นสูง

## การนำเข้า Namespace

เพื่อเริ่มต้น ให้นำเข้า namespace ของ Aspose.BarCode ที่จำเป็นเข้าสู่โปรเจกต์ของคุณ:

```csharp
using Aspose.BarCode.Generation;
```

## ขั้นตอนที่ 1: ตั้งค่า Barcode Generator

สร้างอินสแตนซ์ `BarcodeGenerator` ระบุประเภท **Aztec** และให้ข้อมูลที่คุณต้องการเข้ารหัส

```csharp
string path = "Your Directory Path";
System.Console.WriteLine("AztecErrorLevelExample:");

BarcodeGenerator gen = new BarcodeGenerator(EncodeTypes.Aztec, "Åspóse.Barcóde© is a powerful library to generate & recognize 1D & 2D barcodes");
```

> **เคล็ดลับ:** แทนที่ `"Your Directory Path"` ด้วยเส้นทางแบบ absolute หรือ relative ที่คุณมีสิทธิ์เขียนได้

## ขั้นตอนที่ 2: กำหนด X‑Dimension

X‑Dimension ควบคุมความกว้างของโมดูล (พิกเซล) เล็กที่สุดในบาร์โค้ด การตั้งค่าเป็น 4 พิกเซลจะให้ภาพที่ชัดเจนและสแกนได้ง่าย

```csharp
gen.Parameters.Barcode.XDimension.Pixels = 4;
```

## ขั้นตอนที่ 3: เลือกโหมดสัญลักษณ์ Aztec

Aztec รองรับหลายโหมดสัญลักษณ์ การใช้ `FullRange` จะทำให้ไลบรารีเลือกขนาดที่เหมาะสมอัตโนมัติตามข้อมูลและการตั้งค่าการแก้ไขข้อผิดพลาดของคุณ

```csharp
gen.Parameters.Barcode.Aztec.AztecSymbolMode = AztecSymbolMode.FullRange;
```

## ขั้นตอนที่ 4: ตั้งค่าความจุการแก้ไขข้อผิดพลาด

ตอนนี้เราจะปรับระดับการแก้ไขข้อผิดพลาด ในตัวอย่างนี้เราจะสร้างบาร์โค้ดสองชุด—หนึ่งระดับข้อผิดพลาด 5 % และอีกหนึ่งระดับ 50 %—เพื่อแสดงความแตกต่างของภาพ

```csharp
// Set error correction capacity to 5%
gen.Parameters.Barcode.Aztec.AztecErrorLevel = 5;
gen.Save($"{path}AztecErrorLevel5.png", BarCodeImageFormat.Png);

// Set error correction capacity to 50%
gen.Parameters.Barcode.Aztec.AztecErrorLevel = 50;
gen.Save($"{path}AztecErrorLevel50.png", BarCodeImageFormat.Png);
```

การรันโค้ดจะสร้างไฟล์ PNG สองไฟล์ในโฟลเดอร์ที่ระบุ เวอร์ชัน 50 % มีข้อมูลซ้ำมากกว่า ทำให้ทนต่อความเสียหายได้ดีกว่าแม้ว่าจะทำให้สัญลักษณ์ใหญ่ขึ้นเล็กน้อย
