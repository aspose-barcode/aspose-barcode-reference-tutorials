# การกำหนดค่าอัตราส่วนกว้าง‑แคบของบาร์โค้ดมิติเดียว

คุณกำลังมองหา **วิธีสร้างบาร์โค้ด** อย่างง่ายดายในแอปพลิเคชัน .NET ของคุณหรือไม่? Aspose.BarCode for .NET ทำให้การสร้างบาร์โค้ดในโครงการ Visual Studio ง่ายและทรงพลัง ในบทแนะนำนี้เราจะอธิบายการสร้างบาร์โค้ดมิติเดียวด้วยอัตราส่วนกว้าง‑แคบที่กำหนดเอง, อธิบายว่าทำไมอัตราส่วนนี้ถึงสำคัญ, และแสดงวิธีปรับแต่งสำหรับสภาพแวดล้อมการสแกนที่แตกต่างกัน

## คำตอบอย่างรวดเร็ว
- **อัตราส่วนกว้าง‑แคบควบคุมอะไร?** มันกำหนดความกว้างสัมพัทธ์ของบาร์ “กว้าง” เทียบกับบาร์ “แคบ” ในบาร์โค้ดมิติเดียว  
- **ประเภทบาร์โค้ดที่ใช้ในตัวอย่างคืออะไร?** Code 39 Extended, สัญลักษณ์ที่นิยมสำหรับข้อมูลอักขระและตัวเลข  
- **ฉันสามารถเปลี่ยนอัตราส่วนในขณะรันไทม์ได้หรือไม่?** ได้ – เพียงตั้งค่า `gen.Parameters.Barcode.WideNarrowRatio` เป็นค่าที่ต้องการก่อนบันทึก  
- **ฉันต้องมีใบอนุญาตสำหรับฟีเจอร์นี้หรือไม่?** อัตราส่วนกว้าง‑แคบทำงานได้กับรุ่นทดลองฟรี; ใบอนุญาตเต็มจะเปิดใช้งานตัวเลือกการเรนเดอร์ทั้งหมด  
- **ฟีเจอร์นี้เข้ากันได้กับ .NET Core หรือไม่?** แน่นอน – Aspose.BarCode รองรับ .NET Framework, .NET Core, และ .NET 5/6+

## อัตราส่วนกว้าง‑แคบคืออะไร?
ในบาร์โค้ดมิติเดียวแต่ละบาร์จะเป็น “กว้าง” หรือ “แคบ” เท่านั้น อัตราส่วน (เช่น 2 หรือ 5) กำหนดว่าบาร์กว้างกว้างกว่าบาร์แคบกี่เท่า การปรับอัตราส่วนนี้สามารถเพิ่มความอ่านได้บนเครื่องพิมพ์หรือสแกนเนอร์ความละเอียดต่ำ

## ทำไมต้องใช้ Aspose.BarCode for .NET?
* **การควบคุมเต็มรูปแบบ** – ทุกแง่มุมของภาพรวม, รวมถึงอัตราส่วนกว้าง‑แคบ, สามารถตั้งค่าได้โดยโปรแกรม  
* **ข้ามแพลตฟอร์ม** – ทำงานใน Visual Studio, Visual Studio Code, และรันไทม์ .NET ใดก็ได้  
* **ไม่มีการพึ่งพาภายนอก** – ไม่จำเป็นต้องใช้ DLL เนทีฟหรือเครื่องมือของบุคคลที่สาม  
* **เอกสารและการสนับสนุนที่ครบถ้วน** – มีตัวอย่าง, ฟอรั่ม, และคู่มือเริ่มต้นอย่างรวดเร็ว

## นำเข้า Namespaces

ขั้นแรก, คุณต้องรวม Namespaces ที่จำเป็นในโครงการของคุณเพื่อเข้าถึงฟีเจอร์ของ Aspose.BarCode for .NET คุณสามารถทำได้โดยเพิ่มคำสั่ง using ด้านล่างนี้ที่ส่วนหัวของโค้ดของคุณ:

```csharp
using Aspose.BarCode;
using Aspose.BarCode.Generation;
```

## ขั้นตอนที่ 1: กำหนดเส้นทางไดเรกทอรีของคุณ

เริ่มต้นโดยกำหนดเส้นทางที่คุณต้องการบันทึกรูปภาพบาร์โค้ดที่สร้างขึ้น คุณสามารถทำได้ด้วยโค้ดต่อไปนี้:

```csharp
string path = "Your Directory Path";
```

แทนที่ `"Your Directory Path"` ด้วยเส้นทางไดเรกทอรีจริงที่คุณต้องการบันทึกรูปภาพบาร์โค้ด

## ขั้นตอนที่ 2: สร้างบาร์โค้ดมิติเดียวแบบอัตราส่วนกว้าง‑แคบ

ตอนนี้, เรามาสร้างบาร์โค้ดมิติเดียวด้วยการกำหนดค่าอัตราส่วนกว้าง‑แคบโดยใช้ Aspose.BarCode for .NET ในตัวอย่างนี้เราจะใช้ประเภทการเข้ารหัส Code39Extended และตั้งค่าข้อมูลเป็น `"ASPOSE"`:

```csharp
BarcodeGenerator gen = new BarcodeGenerator(EncodeTypes.Code39Extended, "ASPOSE");
```

บรรทัดโค้ดนี้จะเริ่มต้นตัวสร้างบาร์โค้ดด้วยประเภทการเข้ารหัสและข้อมูลที่ระบุ

## ขั้นตอนที่ 3: ตั้งค่าอัตราส่วนกว้าง/แคบ

อัตราส่วนกว้าง‑แคบกำหนดอัตราส่วนระหว่างองค์ประกอบกว้างและแคบในบาร์โค้ด ในขั้นตอนนี้เราจะตั้งค่าอัตราส่วนกว้าง‑แคบเป็น **2**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 2;
```

จากนั้นเราจะบันทึกรูปภาพบาร์โค้ดที่สร้างขึ้นไปยังเส้นทางที่ระบุ:

```csharp
gen.Save($"{path}WideNarrow2Code39.png", BarCodeImageFormat.Png);
```

## ขั้นตอนที่ 4: ปรับอัตราส่วนกว้าง‑แคบ

คุณสามารถทดลองกับอัตราส่วนกว้าง‑แคบที่ต่างกันเพื่อให้ได้ลักษณะบาร์โค้ดที่ต้องการ ตัวอย่างเช่น หากต้องการบาร์โค้ดที่กว้างขึ้น ให้ตั้งค่าอัตราส่วนกว้าง‑แคบเป็น **5**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 5;
```

และบันทึกรูปภาพบาร์โค้ด:

```csharp
gen.Save($"{path}WideNarrow5Code39.png", BarCodeImageFormat.Png);
```

ตอนนี้คุณได้สร้างบาร์โค้ดมิติเดียวด้วยอัตราส่วนกว้าง‑แคบที่แตกต่างกันโดยใช้ Aspose.BarCode for .NET อย่างสำเร็จ ไลบรารีนี้มอบความยืดหยุ่นให้คุณสร้างบาร์โค้ดที่ปรับให้ตรงกับความต้องการเฉพาะของคุณ

## ปัญหาทั่วไปและวิธีแก้
- **รูปภาพไม่ถูกบันทึก** – ตรวจสอบว่า ตัวแปร `path` ชี้ไปยังโฟลเดอร์ที่มีอยู่และแอปพลิเคชันของคุณมีสิทธิ์เขียน  
- **บาร์โค้ดแสดงผลบิดเบี้ยว** – ลองใช้อัตราส่วนที่ต่ำกว่า (เช่น 2) สำหรับเครื่องพิมพ์ความละเอียดต่ำ; อัตราส่วนที่สูงอาจทำให้เกิดข้อบกพร่องในการพิมพ์  
- **อักขระที่ไม่รองรับ** – Code 39 Extended รองรับชุด ASCII เต็มรูปแบบ; ตรวจสอบให้แน่ใจว่าข้อความข้อมูลของคุณไม่มีอักขระควบคุมที่ห้ามใช้
