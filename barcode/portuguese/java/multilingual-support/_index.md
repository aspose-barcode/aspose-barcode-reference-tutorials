## O que é PDF417 multilíngue Java?
PDF417 é um formato de código de barras linear empilhado que pode codificar grandes quantidades de dados, incluindo caracteres Unicode. Quando um código de barras PDF417 contém caracteres de idiomas como chinês ou turco, você precisa de um decodificador que compreenda Unicode e mapeie esses glifos corretamente. A API Aspose.BarCode para Java oferece suporte multilíngue embutido, permitindo ler e interpretar esses caracteres sem configuração adicional.

## Problemas comuns e soluções
- **Renderização incorreta de caracteres** – Verifique se a imagem de origem tem alta resolução e se o `BarcodeReader` está configurado com `BarcodeReadType.PDF417`.  
- **Licença não encontrada** – Certifique‑se de que o arquivo `Aspose.BarCode.lic` está no classpath ou definido programaticamente via `License.setLicense()`.  
- **Desempenho em lotes grandes** – Use `BarcodeReader.setReadQuality(BarcodeReadQuality.Low)` para processamento mais rápido quando a precisão perfeita não for crítica.

## Perguntas Frequentes

**Q: Posso decodificar códigos de barras PDF417 que misturam vários idiomas em um único símbolo?**  
A: Sim. O mecanismo Unicode do Aspose.BarCode lida automaticamente com dados de múltiplos idiomas.

**Q: Existe um limite para a quantidade de caracteres que posso decodificar?**  
A: PDF417 pode armazenar até 1.850 caracteres de texto; a biblioteca retornará a string completa decodificada, contanto que o código de barras não esteja danificado.

**Q: Preciso definir opções especiais para caracteres turcos?**  
A: Não são necessárias opções especiais; o tratamento padrão de Unicode cobre os caracteres turcos imediatamente.

**Q: Como lidar com arquivos PDF protegidos por senha que contêm códigos de barras PDF417?**  
A: Carregue o PDF com a senha apropriada usando a sobrecarga do `BarCodeReader` que aceita um objeto `PdfDocument`.

**Q: A biblioteca é compatível com Android?**  
A: Sim, o Aspose.BarCode para Java funciona em projetos Android Studio com nível de API mínimo 21.
