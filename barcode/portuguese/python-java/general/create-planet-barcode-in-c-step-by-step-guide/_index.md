# Crie Código de Barras Planet em C# – Guia Passo a Passo

Já precisou **criar código de barras planet** mas não sabia por onde começar? Neste tutorial você verá exatamente **como gerar código de barras planet** usando a biblioteca Aspose.Barcode, com opções de altura padrão e personalizada.

Vamos percorrer todo o processo — desde a instalação do pacote NuGet até o ajuste da altura das barras e a gravação de arquivos PNG. Ao final, você terá quatro imagens prontas para uso: dois códigos de barras Planet (auto e altura de 100 pixels) e dois códigos de barras RM4SCC para comparação. Sem enrolação, apenas código prático que pode ser inserido em qualquer projeto .NET.

## Pré‑requisitos

Antes de começarmos, certifique‑se de que você tem:

* .NET 6.0 ou superior (o código também funciona no .NET Framework 4.7+)
* Visual Studio 2022 (ou qualquer IDE de sua preferência)
* O pacote NuGet **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Uma pasta no disco onde os arquivos PNG gerados possam ser gravados (substitua `YOUR_DIRECTORY` no código).

Se algum desses itens lhe for desconhecido, não se preocupe — instalar um pacote NuGet é tão simples quanto clicar com o botão direito no projeto → *Manage NuGet Packages* → pesquisar “Aspose.Barcode” → *Install*.

## Etapa 1 – Crie um Projeto de Console Minimalista

Primeiro, vamos criar rapidamente um pequeno aplicativo de console para executar o código de barras isoladamente.

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Dica:** Mantenha o método `Main` organizado; delegue o trabalho pesado para métodos separados. Isso facilita o teste e a leitura do código posteriormente.

## Etapa 2 – **Criar Código de Barras Planet** com Altura Padrão

Agora vamos realmente **criar código de barras planet** usando a altura padrão (calculada automaticamente). O `XDimension` controla a largura de um único módulo, enquanto a altura das barras fica a cargo da biblioteca.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Por que isso funciona?*  
`EncodeTypes.Planet` indica ao Aspose que queremos a simbologia Planet, e `"123456"` é a carga de dados. Ao não definir `BarHeight`, o motor escolhe uma altura que satisfaz a especificação automaticamente.

### Saída esperada

Abra `PlanetBarHeightAuto.png` e você verá um código de barras Planet limpo, com aproximadamente 30 mm de altura (o tamanho exato depende do DPI do gravador PNG). Nenhum ajuste manual de altura é necessário.

## Etapa 3 – **Como Gerar Código de Barras Planet** com Altura Personalizada de 100 Pixels

Se o seu design exige uma altura fixa — digamos, 100 pixels — basta definir a propriedade `BarHeight` antes de salvar.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Observe a linha extra `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Ela força o código de barras a esticar (ou encolher) para exatamente 100 pixels de altura, independentemente do comprimento dos dados.

> **Atenção:** Definir uma altura muito pequena pode fazer as barras se fundirem, tornando o código de barras ilegível. Uma regra prática segura é manter a altura pelo menos três vezes a X‑dimension.

## Etapa 4 – Gere Códigos de Barras RM4SCC para Comparação

Às vezes é útil comparar o Planet com outras simbologias postais. RM4SCC é outro formato comum; o código abaixo replica as etapas que usamos para o Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Executar o programa completo agora produz **quatro** arquivos PNG:

| Arquivo | Simbologia | Altura |
|---------|------------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Você pode abrir cada imagem em qualquer visualizador para confirmar as dimensões e a qualidade visual.

## Etapa 5 – Verifique os Códigos de Barras Programaticamente (Opcional)

Se quiser ter certeza de que os códigos de barras gerados são legíveis, o Aspose também oferece um componente **reader**. Aqui está uma verificação rápida que garante que os dados codificados correspondem à nossa string original.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Executar `VerifyBarcodes()` imprime `Decoded value: 123456`, confirmando que a altura personalizada não quebrou a codificação.

## Armadilhas Comuns & Dicas

| Problema | Por que Acontece | Solução |
|----------|------------------|---------|
| O código de barras parece muito fino | X‑dimension definida muito baixa (ex.: 1 pixel) | Aumente `XDimension.Pixels` para 3‑5 na maioria das impressoras |
| A imagem está borrada | Salvando como PNG com DPI padrão (96) em telas de alta resolução | Use `ImageOptions` para definir `ResolutionX` e `ResolutionY` para 300 DPI |
| Altura não foi aplicada | `BarHeight` definido *depois* de chamar `Save()` | Sempre defina `BarHeight` **antes** de `Save()` |
| Caminho do arquivo inválido | `YOUR_DIRECTORY` não existe | Crie a pasta programaticamente: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
