# Maak Planet‑barcode in C# – Stapsgewijze gids

Heb je ooit een **planet barcode** moeten **maken**, maar wist je niet waar te beginnen? In deze tutorial zie je precies **hoe je een planet barcode** kunt genereren met de Aspose.Barcode‑bibliotheek, met zowel standaard‑ als aangepaste hoogte‑opties.

We lopen het volledige proces door—van het installeren van het NuGet‑pakket tot het aanpassen van de balkhoogte en het opslaan van PNG‑bestanden. Aan het einde heb je vier kant‑klaar afbeeldingen: twee Planet‑barcodes (auto en 100‑pixel hoogte) en twee RM4SCC‑barcodes ter vergelijking. Geen poespas, alleen praktische code die je in elk .NET‑project kunt gebruiken.

## Vereisten

* .NET 6.0 of later (de code werkt ook op .NET Framework 4.7+)
* Visual Studio 2022 (of elke IDE die je verkiest)
* Het **Aspose.Barcode for .NET** NuGet‑pakket  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* Een map op schijf waar de gegenereerde PNG‑bestanden kunnen worden weggeschreven (vervang `YOUR_DIRECTORY` in de code).

Als een van deze je onbekend voorkomt, maak je geen zorgen—een NuGet‑pakket installeren is net zo eenvoudig als met de rechtermuisknop op het project klikken → *Manage NuGet Packages* → zoeken op “Aspose.Barcode” → *Install*.

## Stap 1 – Maak een minimaal console‑project

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **Pro tip:** Houd de `Main`‑methode overzichtelijk; de zware taken delegeren aan aparte methoden. Dit maakt de code makkelijker te testen en later te lezen.

## Stap 2 – **Create Planet Barcode** met standaardhoogte

Nu **maken we een planet barcode** met de standaard (automatisch berekende) hoogte. De `XDimension` bepaalt de breedte van één module, terwijl de balkhoogte aan de bibliotheek wordt overgelaten.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*Waarom werkt dit?*  
`EncodeTypes.Planet` vertelt Aspose dat we de Planet‑symbologie willen, en `"123456"` is de gegevenspayload. Door `BarHeight` niet in te stellen, kiest de engine automatisch een hoogte die aan de specificatie voldoet.

### Verwachte output

Open `PlanetBarHeightAuto.png` en je zou een nette Planet‑barcode moeten zien, ongeveer 30 mm hoog (de exacte grootte hangt af van de DPI van de PNG‑writer). Handmatig aanpassen van de hoogte is niet nodig.

## Stap 3 – **How to Generate Planet Barcode** met een aangepaste hoogte van 100 pixel

Als je ontwerp een vaste hoogte vereist—bijvoorbeeld 100 pixels—stel je gewoon de `BarHeight`‑eigenschap in vóór het opslaan.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

Let op de extra regel `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. Deze dwingt de barcode om zich uit te rekken (of te krimpen) tot precies 100 pixels hoog, ongeacht de lengte van de gegevens.

> **Let op:** Een te kleine hoogte kan ervoor zorgen dat de balken samensmelten, waardoor de barcode onleesbaar wordt. Een veilige vuistregel is om de hoogte minstens drie keer de X‑dimension te houden.

## Stap 4 – Genereer RM4SCC‑barcodes ter vergelijking

Soms moet je Planet vergelijken met andere post‑symbologieën. RM4SCC is een ander veelvoorkomend formaat; de onderstaande code spiegelt de stappen die we voor Planet gebruikten.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

Het uitvoeren van het volledige programma levert nu **vier** PNG‑bestanden op:

| Bestand | Symbooltype | Hoogte |
|---------|-------------|--------|
| `PlanetBarHeightAuto.png` | Planet | Auto |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | Auto |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

Je kunt elke afbeelding openen in een willekeurige viewer om de afmetingen en visuele kwaliteit te bevestigen.

## Stap 5 – Verifieer de barcodes programmatisch (optioneel)

Als je extra zeker wilt zijn dat de gegenereerde barcodes scanbaar zijn, biedt Aspose ook een **reader**‑component. Hier is een snelle controle dat de gecodeerde gegevens overeenkomen met onze oorspronkelijke string.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

Het uitvoeren van `VerifyBarcodes()` geeft `Decoded value: 123456` weer, wat bevestigt dat de aangepaste hoogte de codering niet heeft verbroken.

## Veelvoorkomende valkuilen & tips

| Probleem | Waarom het gebeurt | Oplossing |
|----------|--------------------|-----------|
| Barcode lijkt te dun | X‑dimension te laag ingesteld (bijv. 1 pixel) | Verhoog `XDimension.Pixels` naar 3‑5 voor de meeste printers |
| Afbeelding is onscherp | Opslaan als PNG met standaard DPI (96) op schermen met hoge resolutie | Gebruik `ImageOptions` om `ResolutionX` en `ResolutionY` in te stellen op 300 DPI |
| Hoogte niet toegepast | `BarHeight` ingesteld *na* het aanroepen van `Save()` | Stel `BarHeight` altijd **voor** `Save()` in |
| Bestandspad ongeldig | `YOUR_DIRECTORY` bestaat niet | Maak de map programmatisch aan: `Directory.CreateDirectory("YOUR_DIRECTORY");` |
