# 在 C# 中创建 Planet 条形码 – 步骤指南

是否曾需要**创建 planet 条形码**却不知从何入手？在本教程中，你将看到如何使用 Aspose.Barcode 库**生成 planet 条形码**，包括默认高度和自定义高度两种方式。

我们将完整演示整个过程——从安装 NuGet 包到调整条码高度并保存为 PNG 文件。完成后，你将拥有四张可直接使用的图片：两张 Planet 条形码（自动高度和 100 像素高度）以及两张用于对比的 RM4SCC 条形码。内容简洁实用，代码可直接复制到任意 .NET 项目中使用。

## 前置条件

在开始之前，请确保你具备以下环境：

* .NET 6.0 或更高版本（代码同样适用于 .NET Framework 4.7+）
* Visual Studio 2022（或你喜欢的任意 IDE）
* **Aspose.Barcode for .NET** NuGet 包  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* 磁盘上有一个文件夹用于写入生成的 PNG 文件（代码中的 `YOUR_DIRECTORY` 需要替换为实际路径）。

如果上述任意项对你来说陌生，也别担心——安装 NuGet 包只需右键项目 → *Manage NuGet Packages* → 搜索 “Aspose.Barcode” → *Install*。

## 第一步 – 创建最小化控制台项目

首先，搭建一个小型控制台应用，以便在独立环境中运行条码代码。

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **小技巧：** 保持 `Main` 方法简洁；将繁重的逻辑委托给独立的方法。这会让代码更易于测试和阅读。

## 第二步 – 使用默认高度**创建 Planet 条形码**

现在我们使用默认（自动计算）高度**创建 planet 条形码**。`XDimension` 控制单个模块的宽度，条码高度则交由库自行决定。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*为什么这样可以工作？*  
`EncodeTypes.Planet` 告诉 Aspose 使用 Planet 符号，`"123456"` 是数据负载。未设置 `BarHeight` 时，引擎会自动选取满足规范的高度。

### 预期输出

打开 `PlanetBarHeightAuto.png`，你应看到一张约 30 mm 高的清晰 Planet 条形码（具体尺寸取决于 PNG 写入器的 DPI）。无需手动调整高度。

## 第三步 – 使用自定义 100 像素高度**生成 Planet 条形码**

如果你的设计要求固定高度，例如 100 像素，只需在保存前设置 `BarHeight` 属性。

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

注意额外的代码行 `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`。这会强制条码无论数据长度如何，都拉伸（或压缩）到恰好 100 像素高。

> **注意：** 将高度设得过小会导致条纹合并，使条码无法读取。一个安全的经验法则是高度至少为 X‑dimension 的三倍。

## 第四步 – 生成 RM4SCC 条形码进行对比

有时需要将 Planet 与其他邮政符号进行比较。RM4SCC 是另一常见格式，下面的代码与 Planet 的步骤相同。

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

运行完整程序后会生成**四**个 PNG 文件：

| 文件 | 符号类型 | 高度 |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | 自动 |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | 自动 |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

你可以在任意查看器中打开每张图片，确认尺寸和视觉质量。

## 第五步 – 以编程方式验证条形码（可选）

如果想进一步确认生成的条码可被扫描，Aspose 还提供**读取器**组件。下面的代码快速检查编码的数据是否与原始字符串匹配。

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

运行 `VerifyBarcodes()` 会输出 `Decoded value: 123456`，证明自定义高度并未破坏编码。

## 常见问题与技巧

| 问题 | 成因 | 解决方案 |
|-------|----------------|-----|
| 条码显得太细 | X‑dimension 设置过低（如 1 像素） | 将 `XDimension.Pixels` 提升至 3‑5，适配大多数打印机 |
| 图像模糊 | 使用默认 DPI（96）保存 PNG，在高分辨率屏幕上显示 | 使用 `ImageOptions` 将 `ResolutionX` 与 `ResolutionY` 设置为 300 DPI |
| 高度未生效 | 在调用 `Save()` 之后才设置 `BarHeight` | 始终在 **保存前** 设置 `BarHeight` |
| 文件路径无效 | `YOUR_DIRECTORY` 不存在 | 通过代码创建文件夹：`Directory.CreateDirectory("YOUR_DIRECTORY");` |
