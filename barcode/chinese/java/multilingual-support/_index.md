## 什么是多语言 PDF417 条形码 Java？
PDF417 是一种堆叠线性条形码格式，能够编码大量数据，包括 Unicode 字符。当 PDF417 条形码包含中文或土耳其语等语言的字符时，您需要一个能够理解 Unicode 并正确映射这些字形的解码器。Aspose.BarCode Java API 提供内置的多语言支持，使您无需额外配置即可读取和解释这些字符。

## 为什么在多语言 PDF417 条形码 Java 中使用 Aspose.BarCode？

- **跨平台** – 在 Windows、Linux 和 macOS 上使用相同代码库即可运行。

## 常见问题及解决方案
- **字符渲染不正确** – 确认源图像为高分辨率，并且 `BarcodeReader` 已配置为 `BarcodeReadType.PDF417`。  
- **未找到许可证** – 确保 `Aspose.BarCode.lic` 文件放置在类路径中，或通过 `License.setLicense()` 以编程方式设置。  
- **大批量处理性能** – 当对精度要求不高时，可使用 `BarcodeReader.setReadQuality(BarcodeReadQuality.Low)` 以加快处理速度。  

## 常见问题

**问：我可以解码在单个符号中混合多种语言的 PDF417 条形码吗？**  
A: 是的。Aspose.BarCode 的 Unicode 引擎会自动处理混合语言数据。

**问：我可以解码的字符数量有限制吗？**  
A: PDF417 最多可存储 1,850 个文本字符；只要条形码未受损，库会返回完整的解码字符串。

**问：我需要为土耳其字符设置任何特殊选项吗？**  
A: 不需要特殊选项；默认的 Unicode 处理已开箱即支持土耳其字符。

**问：如何处理包含 PDF417 条形码的受密码保护的 PDF 文件？**  
A: 使用接受 `PdfDocument` 对象的 `BarCodeReader` 重载，并提供相应的密码来加载 PDF。

**问：该库兼容 Android 吗？**  
A: 是的，Aspose.BarCode for Java 可在 Android Studio 项目中使用，最低 API 级别为 21。
