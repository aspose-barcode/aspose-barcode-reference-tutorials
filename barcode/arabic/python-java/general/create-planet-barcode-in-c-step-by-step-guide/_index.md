# إنشاء باركود كوكب في C# – دليل خطوة‑بخطوة

هل احتجت يوماً إلى **إنشاء باركود كوكب** لكن لم تكن متأكدًا من أين تبدأ؟ في هذا الدرس ستتعرف بالضبط على **كيفية إنشاء باركود كوكب** باستخدام مكتبة Aspose.Barcode، مع خيارات الارتفاع الافتراضي والمخصص.

سنستعرض العملية بالكامل—من تثبيت حزمة NuGet إلى تعديل ارتفاع الخطوط وحفظ ملفات PNG. في النهاية ستحصل على أربع صور جاهزة للاستخدام: باركودان من نوع Planet (ارتفاع تلقائي وارتفاع 100 بكسل) وباركودان من نوع RM4SCC للمقارنة. لا إطالة، فقط كود عملي يمكنك إدراجه في أي مشروع .NET.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من وجود التالي:

* .NET 6.0 أو أحدث (الكود يعمل على .NET Framework 4.7+ أيضاً)
* Visual Studio 2022 (أو أي بيئة تطوير تفضلها)
* حزمة NuGet **Aspose.Barcode for .NET**  
  ```bash
  dotnet add package Aspose.Barcode
  ```
* مجلد على القرص حيث يمكن كتابة ملفات PNG المُولدة (استبدل `YOUR_DIRECTORY` في الكود).

إذا كان أي من ذلك غير مألوف لك، لا تقلق—تثبيت حزمة NuGet سهل مثل النقر بزر الماوس الأيمن على المشروع → *Manage NuGet Packages* → البحث عن “Aspose.Barcode” → *Install*.

## الخطوة 1 – إنشاء مشروع وحدة تحكم بسيط

```csharp
using System;
using Aspose.BarCode.Generation;

namespace PlanetBarcodeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll call helper methods here – see the sections below.
            GeneratePlanetBarcodes();
            GenerateRm4sccBarcodes();
            Console.WriteLine("All barcodes generated successfully.");
        }

        // Helper methods are defined later.
    }
}
```

> **نصيحة احترافية:** حافظ على تنظيم طريقة `Main`؛ وزع الأعمال الثقيلة على طرق منفصلة. هذا يجعل الكود أسهل للاختبار والقراءة لاحقًا.

## الخطوة 2 – **إنشاء باركود كوكب** مع الارتفاع الافتراضي

الآن سنقوم فعليًا **بإنشاء باركود كوكب** باستخدام الارتفاع الافتراضي (المُحسب تلقائيًا). المتغير `XDimension` يتحكم في عرض الوحدة الواحدة، بينما يُترك ارتفاع الخطوط للمكتبة.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4; // narrow modules
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);
}
```

*لماذا يعمل هذا؟*  
`EncodeTypes.Planet` يخبر Aspose أننا نريد رمز Planet، و `"123456"` هو البيانات المشفرة. بعدم ضبط `BarHeight`، يختار المحرك ارتفاعًا يطابق المواصفات تلقائيًا.

### النتيجة المتوقعة

افتح `PlanetBarHeightAuto.png` وسترى باركود Planet نظيف، بارتفاع تقريبًا 30 مم (الحجم الدقيق يعتمد على DPI الخاص بكاتب PNG). لا حاجة لتعديل الارتفاع يدويًا.

## الخطوة 3 – **كيفية إنشاء باركود كوكب** بارتفاع مخصص 100‑بكسل

إذا كان تصميمك يتطلب ارتفاعًا ثابتًا—مثلاً 100 بكسل—فما عليك سوى ضبط خاصية `BarHeight` قبل الحفظ.

```csharp
static void GeneratePlanetBarcodes()
{
    // 1️⃣ Default height (already shown)
    var planetDefault = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetDefault.Parameters.Barcode.XDimension.Pixels = 4;
    planetDefault.Save("YOUR_DIRECTORY/PlanetBarHeightAuto.png", BarCodeImageFormat.Png);

    // 2️⃣ Custom 100‑pixel height
    var planetCustom = new BarcodeGenerator(EncodeTypes.Planet, "123456");
    planetCustom.Parameters.Barcode.XDimension.Pixels = 4;
    planetCustom.Parameters.Barcode.BarHeight.Pixels = 100; // explicit height
    planetCustom.Save("YOUR_DIRECTORY/PlanetBarHeight100.png", BarCodeImageFormat.Png);
}
```

لاحظ السطر الإضافي `planetCustom.Parameters.Barcode.BarHeight.Pixels = 100;`. هذا يجبر الباركود على التمدد (أو الانكماش) إلى ارتفاع 100 بكسل بالضبط، بغض النظر عن طول البيانات.

> **احذر:** ضبط ارتفاع صغير جدًا قد يتسبب في دمج الخطوط، مما يجعل الباركود غير قابل للقراءة. القاعدة العامة الآمنة هي الحفاظ على ارتفاع لا يقل عن ثلاثة أضعاف X‑dimension.

## الخطوة 4 – إنشاء باركودات RM4SCC للمقارنة

أحيانًا تحتاج إلى مقارنة Planet مع رموز بريدية أخرى. RM4SCC هو تنسيق شائع آخر؛ الكود أدناه يعكس الخطوات التي استخدمناها لـ Planet.

```csharp
static void GenerateRm4sccBarcodes()
{
    // Default height
    var rm4sccDefault = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccDefault.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccDefault.Save("YOUR_DIRECTORY/RM4SCCBarHeightAuto.png", BarCodeImageFormat.Png);

    // Custom 100‑pixel height
    var rm4sccCustom = new BarcodeGenerator(EncodeTypes.RM4SCC, "123456");
    rm4sccCustom.Parameters.Barcode.XDimension.Pixels = 4;
    rm4sccCustom.Parameters.Barcode.BarHeight.Pixels = 100;
    rm4sccCustom.Save("YOUR_DIRECTORY/RM4SCCBarHeight100.png", BarCodeImageFormat.Png);
}
```

تشغيل البرنامج الكامل الآن ينتج **أربع** ملفات PNG:

| الملف | الرمز | الارتفاع |
|------|-----------|--------|
| `PlanetBarHeightAuto.png` | Planet | تلقائي |
| `PlanetBarHeight100.png` | Planet | 100 px |
| `RM4SCCBarHeightAuto.png` | RM4SCC | تلقائي |
| `RM4SCCBarHeight100.png` | RM4SCC | 100 px |

يمكنك فتح كل صورة في أي عارض لتأكيد الأبعاد والجودة البصرية.

## الخطوة 5 – التحقق من الباركودات برمجيًا (اختياري)

إذا أردت التأكد تمامًا من أن الباركودات المُولدة قابلة للمسح، توفر Aspose أيضًا مكوّن **القارئ**. إليك فحص سريع للتأكد من أن البيانات المشفرة تطابق السلسلة الأصلية.

```csharp
using Aspose.BarCode.Reader;

static void VerifyBarcodes()
{
    var reader = new BarCodeReader("YOUR_DIRECTORY/PlanetBarHeight100.png", DecodeType.Planet);
    foreach (BarCodeResult result in reader.ReadBarCodes())
    {
        Console.WriteLine($"Decoded value: {result.CodeText}");
    }
}
```

تشغيل `VerifyBarcodes()` يطبع `Decoded value: 123456`، مؤكدًا أن ارتفاع مخصص لم يُفسد الترميز.

## المشكلات الشائعة والنصائح

| المشكلة | السبب | الحل |
|-------|----------------|-----|
| الباركود يبدو رفيعًا جدًا | تم ضبط X‑dimension منخفضًا جدًا (مثلاً 1 بكسل) | زيادة `XDimension.Pixels` إلى 3‑5 لمعظم الطابعات |
| الصورة غير واضحة | الحفظ كـ PNG بدقة DPI افتراضية (96) على شاشات عالية الدقة | استخدام `ImageOptions` لتعيين `ResolutionX` و `ResolutionY` إلى 300 DPI |
| الارتفاع غير مطبق | `BarHeight` تم ضبطه *بعد* استدعاء `Save()` | دائمًا اضبط `BarHeight` **قبل** `Save()` |
| مسار الملف غير صالح | `YOUR_DIRECTORY` غير موجود | إنشاء المجلد برمجيًا: `Directory.CreateDirectory("YOUR_DIRECTORY");` |

## الخلاصة

لقد **أنشأنا صور باركود كوكب** من الصفر، واستكشفنا **كيفية إنشاء باركود كوكب** بارتفاعين تلقائي ومحدد، وحتى قارنّاها جنبًا إلى جنب مع ما يعادلها من RM4SCC. المثال الكامل القابل للتنفيذ موجود في ملف `Program.cs` واحد، لذا يمكنك نسخه ولصقه في أي مشروع .NET والبدء بطباعة الباركودات فورًا.

### ما التالي؟

* جرّب قيمًا مختلفة لـ `XDimension` لترى كيف يؤثر عرض الوحدة على موثوقية المسح.  
* جرب إخراجًا ملونًا بتعيين `ImageOptions.ForeColor` و `BackColor`.  
* استكشف رموز بريدية أخرى (مثل **Postnet**، **Intelligent Mail**) باستخدام نفس النمط.  

هل لديك أسئلة حول حالات خاصة—مثل إنشاء باركود كوكب لحمل 12 رقمًا أو تضمين PNG داخل PDF؟ اترك تعليقًا، وسنستكشف تلك السيناريوهات معًا.

برمجة سعيدة، ولتكن باركوداتك دائمًا قابلة للمسح من المحاولة الأولى!
