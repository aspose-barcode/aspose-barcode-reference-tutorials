# تكوين نسبة العرض‑الضيق أحادي البُعد

هل تبحث عن **كيفية إنشاء الباركود** بسهولة في تطبيقات .NET الخاصة بك؟ تجعل Aspose.BarCode for .NET توليد الباركود في مشاريع Visual Studio بسيطًا وقويًا. في هذا البرنامج التعليمي سنستعرض إنشاء باركودات أحادية البُعد بنسبة عرض‑ضيق مخصصة، نشرح لماذا النسبة مهمة، ونظهر لك كيفية تعديلها لبيئات المسح المختلفة.

## إجابات سريعة
- **ما الذي تتحكم فيه نسبة العرض‑الضيق؟** إنها تحدد العرض النسبي للأشرطة “العريضة” مقارنة بالأشرطة “الضيقة” في باركود أحادي البُعد.  
- **ما نوع الباركود المستخدم في المثال؟** Code 39 Extended، رمزية شائعة للبيانات الحرفية الرقمية.  
- **هل يمكنني تغيير النسبة أثناء التشغيل؟** نعم – فقط قم بتعيين `gen.Parameters.Barcode.WideNarrowRatio` إلى القيمة المطلوبة قبل الحفظ.  
- **هل أحتاج إلى ترخيص لهذه الميزة؟** تعمل نسبة العرض‑الضيق مع النسخة التجريبية المجانية؛ الترخيص الكامل يفتح جميع خيارات العرض.  
- **هل هذا متوافق مع .NET Core؟** بالتأكيد – تدعم Aspose.BarCode .NET Framework و .NET Core و .NET 5/6+.

## ما هي نسبة العرض‑الضيق؟
في الباركودات أحادية البُعد كل شريط يكون إما “عريض” أو “ضيق”. النسبة (مثلاً 2 أو 5) تحدد كم مرة يكون الشريط العريض أوسع من الشريط الضيق. تعديل هذه النسبة يمكن أن يحسن قابلية القراءة على الطابعات أو الماسحات ذات الدقة المنخفضة.

## لماذا نستخدم Aspose.BarCode for .NET؟
* **تحكم كامل** – يمكن ضبط كل جانب بصري، بما في ذلك نسبة العرض‑الضيق، برمجيًا.  
* **متعدد المنصات** – يعمل في Visual Studio و Visual Studio Code وأي بيئة تشغيل .NET.  
* **بدون تبعيات خارجية** – لا حاجة إلى ملفات DLL أصلية أو أدوات طرف ثالث.  
* **توثيق ودعم غني** – يتضمن أمثلة، منتديات، وأدلة بدء سريعة.

## استيراد المساحات الاسمية

أولاً، تحتاج إلى تضمين المساحات الاسمية الضرورية في مشروعك للوصول إلى ميزات Aspose.BarCode for .NET. يمكنك القيام بذلك بإضافة عبارات using التالية في أعلى الكود الخاص بك:

```csharp
using Aspose.BarCode;
using Aspose.BarCode.Generation;
```

## الخطوة 1: تحديد مسار الدليل الخاص بك

ابدأ بتحديد المسار حيث تريد حفظ صور الباركود المولدة. يمكنك القيام بذلك باستخدام الكود التالي:

```csharp
string path = "Your Directory Path";
```

استبدل `"Your Directory Path"` بالمسار الفعلي للدليل حيث تريد حفظ صور الباركود.

## الخطوة 2: إنشاء باركود أحادي البُعد بنسبة عرض‑ضيق

الآن، لننشئ باركودًا أحادي البُعد مع تكوين نسبة عرض‑ضيق باستخدام Aspose.BarCode for .NET. في هذا المثال، سنستخدم نوع الترميز Code39Extended ونحدد البيانات إلى `"ASPOSE"`:

```csharp
BarcodeGenerator gen = new BarcodeGenerator(EncodeTypes.Code39Extended, "ASPOSE");
```

هذا السطر من الكود ي初始化 مولد الباركود بنوع الترميز والبيانات المحددين.

## الخطوة 3: ضبط نسبة العرض/الضيق

نسبة العرض‑الضيق تحدد النسبة بين العناصر العريضة والضيقة في الباركود. في هذه الخطوة، سنضبط نسبة العرض‑الضيق إلى **2**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 2;
```

ثم، سنحفظ صورة الباركود المولدة إلى المسار المحدد:

```csharp
gen.Save($"{path}WideNarrow2Code39.png", BarCodeImageFormat.Png);
```

## الخطوة 4: تعديل نسبة العرض‑الضيق

يمكنك تجربة نسب عرض‑ضيق مختلفة للحصول على المظهر المطلوب للباركود. على سبيل المثال، إذا أردت باركودًا أوسع، اضبط نسبة العرض‑الضيق إلى **5**:

```csharp
gen.Parameters.Barcode.WideNarrowRatio = 5;
```

واحفظ صورة الباركود:

```csharp
gen.Save($"{path}WideNarrow5Code39.png", BarCodeImageFormat.Png);
```

- **لم يتم حفظ الصورة** – تحقق من أن المتغير `path` يشير إلى مجلد موجود وأن تطبيقك يمتلك صلاحيات الكتابة.  
- **الباركود يظهر مشوّهًا** – جرّب نسبة أقل (مثلاً 2) للطابعات ذات الدقة المنخفضة؛ النسب الأعلى قد تتسبب في ظهور عيوب طباعة.
