# إنشاء باركود Code128 باستخدام Aspose.Barcode Python – دليل كامل

هل احتجت يوماً إلى **إنشاء باركود Code128** في مشروع Python لكنك لم تكن متأكدًا من كيفية تضمين الأحرف غير ASCII؟ لست وحدك. يواجه العديد من المطورين صعوبة عندما يحاولون خلط البيانات الرقمية مع نص Unicode — خاصة الروسية أو الصينية — داخل باركود واحد. الخبر السار؟ Aspose.Barcode للـ Python يجعل الأمر سهلًا، ومع الاستخدام الصحيح لـ *barcode with ECI* يمكنك الحفاظ على قابلية القراءة للمسّاحين في جميع أنحاء العالم.

في هذا الدرس سنستعرض مثالًا كاملاً قابلاً للتنفيذ يوضح بالضبط كيفية **إنشاء باركود Code128** باستخدام مكتبة Aspose.Barcode Python، وكيفية بناء نص كود موسع يتضمن مقطع ECI، ولماذا كل سطر مهم. في النهاية ستحصل على سكريبت جاهز للتنفيذ، وفهم واضح للمفاهيم الأساسية، وبعض النصائح الاحترافية لتجنب الأخطاء الشائعة.

## ما ستتعلمه

- إعداد **Aspose.Barcode Python** في بيئتك.  
- بناء سلسلة نص كود موسعة تمزج بين الأرقام العادية ونص UTF‑8 عبر مقطع ECI.  
- تكوين `BarCodeGenerator` لإنتاج **باركود Code128**.  
- حفظ الصورة والتحقق من النتيجة.  
- معالجة الحالات الحدية لمجموعات الأحرف المختلفة وأنواع الباركود.

لا حاجة لأي وثائق خارجية — كل ما تحتاجه موجود هنا.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من وجود ما يلي:

1. **Python 3.8+** مثبت (المكتبة تدعم 3.6 وما فوق).  
2. رخصة صالحة لـ **Aspose.Barcode for Python via .NET** أو أنك موافق على علامة التقييم المائية.  
3. إمكانية الوصول إلى الطرفية أو بيئة التطوير المتكاملة حيث يمكنك تثبيت الحزم (`pip`).  
4. مجلد سيتم حفظ ملف PNG الناتج فيه (استبدل `YOUR_DIRECTORY` في الكود).

إذا كان أي من هذه غير مألوف لك، لا تقلق — تثبيت الحزمة أمر بأمر واحد والبقية هي Python بحتة.

## الخطوة 1: تثبيت Aspose.Barcode للـ Python

افتح الطرفية وشغّل الأمر التالي:

```bash
pip install aspose-barcode
```

> **نصيحة احترافية:** استخدم بيئة افتراضية (`python -m venv venv`) لعزل الاعتمادات. هذا يمنع أيضًا تعارض الإصدارات مع مشاريع أخرى.

الحزمة تجلب وقت تشغيل .NET تحت الغطاء، لذا قد يستغرق الاستيراد الأول ثانيةً بينما تُحمَّل المكتبات الأصلية.

---

## الخطوة 2: إنشاء مُنشئ نص كود موسع

قلب دعم **barcode with ECI** يكمن في `ExtCodetextBuilder`. يتيح لك دمج المقاطع العادية مع الأجزاء المشفرة بـ ECI، مما يضمن أن يفسّر المسّاحون النص Unicode بشكل صحيح.

```python
import aspose.barcode as barcode

# Initialize the builder – it starts empty
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()  # Explicitly clear any residual data (good hygiene)
```

**لماذا `clear()`؟**  
إذا أعدت استخدام المُنشئ داخل حلقة، قد تتراكم البيانات وتفسد الباركود التالي. استدعاء `clear()` يضمن بداية نظيفة في كل مرة.

---

## الخطوة 3: إضافة مقطع رقمي عادي

معظم باركودات Code128 تبدأ بمعرف رقمي — فكر في رقم طلب أو معرف منتج. إضافته كمقاطع *عادية* يبقي الترميز بسيطًا ومضغوطًا.

```python
# Plain numeric part – 10 digits in this example
ext_builder.add_plain_codetext("1234567890")
```

الطريقة تختار تلقائيًا مجموعة Code128 المثلى (A أو B أو C) للأرقام، مما ينتج أصغر باركود ممكن.

---

## الخطوة 4: إدراج مقطع ECI بنص روسي

الآن للجزء الصعب: تضمين الأحرف الروسية (`Пример текста`). بدون علامة ECI، سيُفسّر معظم المسّاحين تلك البايتات بشكل غير صحيح، مما ينتج نصًا مشوّهًا.

```python
# ECI value 26 corresponds to UTF‑8 per ISO/IEC 18004
ext_builder.add_eci_codetext(26, "Пример текста")  # UTF‑8 encoded text
```

**ما الذي يحدث؟**  
- `26` يخبر الباركود أن البايتات التالية تستخدم ترميز UTF‑8.  
- المكتبة تتولى التحويل، لذا كل ما عليك هو تمرير سلسلة Unicode.  
- عندما يقرأ المسّاح الباركود، يلاحظ وجود ECI، يتحول إلى UTF‑8، ويعرض النص الروسي بشكل صحيح.

> **سؤال شائع:** *ماذا لو احتجت لغة أخرى؟*  
> استخدم قيمة ECI المناسبة (مثلاً 27 لـ ISO‑8859‑1، 28 لـ ISO‑8859‑2). جدول القيم متوفر في وثائق Aspose، لكن النمط يبقى نفسه.

## الخطوة 5: استرجاع نص الكود الموسع الكامل

بعد إضافة كلا المقاطع، نستخرج السلسلة المدمجة. هذه هي القيمة التي ستُعيّنها إلى خاصية `code_text` للمولد.

```python
# Get the ready‑to‑use extended codetext
extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)
```

الناتج المطبوع يبدو تقريبًا هكذا:

```
Extended codetext: 1234567890[ECI26]Пример текста
```

الرمز `[ECI26]` غير مرئي في الصورة النهائية لكنه أساسي للفك الصحيح.

## الخطوة 6: تكوين BarCodeGenerator لـ Code128

الآن نجمع كل شيء. `BarCodeGenerator` هو العنصر الأساسي الذي يرسم الصورة.

```python
# Initialize generator
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128   # Primary focus: generate Code128 barcode
generator.code_text = extended_code_text          # Feed the extended codetext

# Optional: tweak appearance
generator.parameters.barcode.x_dimension = 0.5   # Narrower bars for higher density
generator.parameters.barcode.bar_height = 30     # Height in points
```

**لماذا نضبط `x_dimension`؟**  
عرض الوحدة الأصغر ينتج باركودًا أكثر تكثيفًا — مفيد عندما تكون المساحة محدودة. عدّل القيمة وفقًا لدقة الطابعة (DPI).

## الخطوة 7: حفظ صورة الباركود

أخيرًا، اكتب ملف PNG إلى القرص. يمكنك أيضًا تصدير إلى JPEG أو BMP أو حتى SVG بتغيير امتداد الملف.

```python
output_path = "YOUR_DIRECTORY/extended_code.png"
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

افتح الصورة بأي عارض؛ يجب أن ترى باركود Code128 نظيف، وعند مسحه بقارئ حديث، سيعيد `1234567890Пример текста`.

## السكريبت الكامل – انسخه وشغّله

فيما يلي السكريبت الكامل المستقل. انسخه إلى ملف باسم `generate_code128.py`، عدّل `YOUR_DIRECTORY`، ثم نفّذ `python generate_code128.py`.

```python
import aspose.barcode as barcode
import os

# ------------------------------------------------------------------
# Step 1: Prepare output directory (creates it if missing)
# ------------------------------------------------------------------
output_dir = "YOUR_DIRECTORY"
os.makedirs(output_dir, exist_ok=True)

# ------------------------------------------------------------------
# Step 2: Build extended codetext (plain + ECI segment)
# ------------------------------------------------------------------
ext_builder = barcode.generation.ExtCodetextBuilder()
ext_builder.clear()
ext_builder.add_plain_codetext("1234567890")
ext_builder.add_eci_codetext(26, "Пример текста")  # 26 = UTF‑8

extended_code_text = ext_builder.get_extended_codetext()
print("Extended codetext:", extended_code_text)

# ------------------------------------------------------------------
# Step 3: Configure generator for Code128
# ------------------------------------------------------------------
generator = barcode.BarCodeGenerator()
generator.symbology = barcode.Symbology.CODE128
generator.code_text = extended_code_text

# Optional appearance tweaks
generator.parameters.barcode.x_dimension = 0.5
generator.parameters.barcode.bar_height = 30

# ------------------------------------------------------------------
# Step 4: Save the barcode image
# ------------------------------------------------------------------
output_path = os.path.join(output_dir, "extended_code.png")
generator.save(output_path)

print(f"Barcode saved to {output_path}")
```

**الناتج المتوقع** (في الطرفية):

```
Extended codetext: 1234567890[ECI26]Пример текста
Barcode saved to YOUR_DIRECTORY/extended_code.png
```

افتح `extended_code.png` وسترى باركود Code128 واضح جاهز للاستخدام الإنتاجي.

## نظرة عميقة: كيف يعمل آلية ECI

ECI (Extended Channel Interpretation) هي جزء من معايير QR والباركود تُشير إلى تغيير في ترميز الأحرف أثناء التدفق. في سياق Code128:

1. يبدأ الباركود في وضع ISO‑8859‑1 الافتراضي.  
2. عندما يصادف المسّاح رمز التحكم ECI (القيمة 26 في مثالنا)، يتحول إلى UTF‑8.  
3. تُفسَّر جميع البايتات التالية كـ UTF‑8 حتى يظهر ECI آخر أو ينتهي البيانات.

بما أننا حددنا صراحةً `add_eci_codetext(26, ...)`، تقوم Aspose بإدراج رموز التحكم الصحيحة خلف الكواليس. إذا حذفت ECI، سيُشفّر النص الروسي باستخدام صفحة الترميز الافتراضية، وعادةً ما ينتج عنه تشويش غير قابل للقراءة على معظم المسّاحين.

## معالجة الحالات الحدية

### 1. مقاطع ECI متعددة

إذا احتجت إلى دمج نص ياباني وعربي، يمكنك استدعاء `add_eci_codetext` عدة مرات بقيم ECI مختلفة. المُنشئ يدمجها بالترتيب الذي تضيفه فيه.

```python
ext_builder.add_eci_codetext(27, "Texto en Español")   # ISO‑8859‑1
ext_builder.add_eci_codetext(28, "文字列日本語")       # ISO‑8859‑2 (example)
```

### 2. حدود الطول

يمكن لـ Code128 ترميز ما يصل إلى 2 023 حرفًا، لكن الحد العملي غالبًا ما يكون أقل بسبب ذاكرة المسّاح. إذا تجاوزت هذا الحد، سيُطلق المولد استثناء `ArgumentException`. لتكون آمنًا، حافظ على إجمالي الطول أقل من 1 000 حرف عند خلط مقاطع ECI.

### 3. التوافق مع المسّاحين القديمة

بعض المسّاحين القديمة تتجاهل رموز التحكم ECI. في مثل هذه البيئات، قد تحتاج إلى توليد باركودات منفصلة — واحدة للجزء الرقمي، أخرى للنص Unicode — أو استخدام نوع رموز آخر مثل QR الذي يدعم ECI على نطاق أوسع.

## نصائح احترافية للاستخدام الإنتاجي

- **تفعيل الرخصة مبكرًا:** النسخة التجريبية تضيف علامة مائية صغيرة. ضع ملف الرخصة قبل أول استدعاء لـ `BarCodeGenerator` لتجنبها.  
- **معالجة دفعات:** عند توليد آلاف الباركودات، أعد استخدام نفس كائن `ExtCodetextBuilder` — فقط استدعِ `clear()` في كل دورة.  
- **سلامة الخيوط:** كل خيط يجب أن يمتلك نسخة خاصة به من `BarCodeGenerator`؛ الكائنات غير آمنة للخلط بين الخيوط.  
- **الأداء:** ضبط `x_dimension` منخفضًا جدًا قد يزيد من زمن الرسم. اختبر الأداء على الأجهزة المستهدفة إذا كنت في سيناريو عالي الإنتاجية.  
- **الاختبار:** دائمًا امسح الباركود المولد باستخدام قارئين مختلفين على الأقل (تطبيق هاتف ومسّاح مادي) للتحقق من معالجة ECI.
