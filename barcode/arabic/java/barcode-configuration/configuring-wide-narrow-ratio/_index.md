# كيفية إنشاء باركود CODE_128 بنسبة العرض‑الضيق في جافا

## المقدمة

إذا كنت بحاجة إلى **إنشاء باركود code128** بنسب بصرية دقيقة، فإن تعديل نسبة العرض‑الضيق هو المفتاح. في هذا الدرس سنستعرض عملية **إنشاء باركود خطوة بخطوة** باستخدام Aspose.BarCode for Java، موضحين لك كيفية ضبط النسبة، **إنشاء صور باركود PNG**، و**حفظ صورة الباركود** على القرص. سواءً كنت تبني ملصقات جرد، بطاقات شحن، أو أي تطبيق يتطلب باركود CODE_128 مُصممًا خصيصًا، ستجد كل ما تحتاجه هنا.

## إجابات سريعة
- **ما هي نسبة العرض‑الضيق؟** تتحكم في العرض النسبي للأشرطة العريضة مقارنةً بالأشرطة الضيقة في الباركود.  
- **أي رموز تدعم ضبط النسبة؟** معظم رموز الباركود أحادية البُعد، بما في ذلك CODE_128، تسمح بتعيين نسبة مخصصة.  
- **هل أحتاج إلى ترخيص؟** يتوفر إصدار تجريبي مجاني، لكن الترخيص التجاري مطلوب للاستخدام في الإنتاج.  
- **هل يمكنني إنشاء صورة باركود بصيغة PNG؟** نعم—استخدم `generator.save(...)` لإنشاء صور باركود PNG.  
- **هل الكود متوافق مع Java 8+؟** بالتأكيد؛ Aspose.BarCode يعمل مع جميع إصدارات جافا الحديثة.

## المتطلبات المسبقة

قبل أن نغوص في الكود، تأكد من توفر ما يلي:

- مجموعة تطوير جافا (JDK) مثبتة على جهازك.  
- مكتبة Aspose.BarCode for Java. حمّلها من [رابط التحميل](https://releases.aspose.com/barcode/java/).

## استيراد الحزم

للبدء، استورد الفئة الأساسية من Aspose.BarCode إلى مشروعك.

```java
// Import Aspose.BarCode library
import com.aspose.barcode.generation.BarcodeGenerator;
```

## ما هي نسبة العرض‑الضيق ولماذا نضبطها؟

نسبة العرض‑الضيق تحدد مدى سمك الأشرطة “العريضة” مقارنةً بالأشرطة “الضيقة”. ضبط هذه النسبة يمكن أن يحسن قابلية القراءة للماسحات الضوئية، يلتزم بمعايير الطباعة المحددة، أو ببساطة يتماشى مع النمط البصري للعلامة التجارية.

## كيفية إنشاء باركود code128 بنسبة العرض‑الضيق في جافا

فيما يلي دليل **باركود خطوة بخطوة** يوضح لك كل جزء من العملية.

### الخطوة 1: تعيين دليل المستندات

```java
// The path to the resource directory.
String dataDir = "Your Document Directory";
```

تأكد من وجود الدليل ولديك صلاحيات كتابة؛ هذا هو المكان الذي سيتم فيه وضع ملف **حفظ صورة الباركود**.

### الخطوة 2: إنشاء كائن الباركود

```java
// Instantiate barcode object
// Create an instance of BarcodeGenerator, specify codetext and symbology in the constructor
BarcodeGenerator generator = new BarcodeGenerator(EncodeTypes.CODE_128, "12345678");
```

هنا نقوم **بإنشاء باركود code128** بتمرير `EncodeTypes.CODE_128` إلى المُنشئ.

### الخطوة 3: ضبط نسبة العرض‑الضيق

```java
// Set the wide to narrow ratio for the barcode
generator.getParameters().getBarcode().setWideNarrowRatio(3.0f);
```

طريقة `setWideNarrowRatio` تتيح لك ضبط التباعد البصري بدقة. القيمة `3.0f` تعني أن الشريط العريض يساوي ثلاثة أضعاف عرض الشريط الضيق.

### الخطوة 4: حفظ الصورة على القرص

```java
// Save the image to disk in PNG format
generator.save(dataDir + "wideNarrowRatio.png");
```

استدعاء `save` سيقوم **بإنشاء صورة باركود** وتخزينها كملف PNG، مُكملًا خطوة **حفظ صورة الباركود**.

## لماذا نضبط نسبة العرض‑الضيق؟

- **توافق الماسحات الضوئية:** بعض الماسحات تفضّل نسبة بين 2.0 و 3.0 للحصول على معدلات قراءة مثالية.  
- **جودة الطباعة:** ضبط النسبة يمكن أن يعوّض عن قيود DPI للطابعة، مما يمنع تشوه الأشرطة.  
- **اتساق العلامة التجارية:** بعض الشركات ترغب في باركود يتماشى بصريًا مع ألوان الشركة أو إرشادات التصميم.

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|-------|--------|-----|
| الباركود يبدو مشوّهًا | نسبة غير مناسبة للطابعة | اضبط القيمة الممررة إلى `setWideNarrowRatio` (مثلاً 2.0‑2.5). |
| الملف غير مُنشأ | مسار `dataDir` غير صالح أو صلاحيات غير كافية | تحقق من مسار الدليل وتأكد من أن التطبيق يملك صلاحية كتابة. |
| الماسح لا يستطيع قراءة الباركود | النسبة خارج النطاق الموصى به للرمز | استخدم نسبًا قياسية (2.0‑3.0) أو اختبر مع الماسح المستهدف. |
